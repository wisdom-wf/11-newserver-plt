# 项目重构方案

## 一、问题诊断

### 1.1 列设置功能失效（核心问题）

**问题现象**：所有列表的列设置都不能生效，修改了3次都修不好

**根本原因**：`TableColumnSetting` 组件期望的数据格式是 `NaiveUI.TableColumnCheck[]`，但业务代码传入的是自定义格式：

```typescript
// 框架期望的格式（正确）
interface TableColumnCheck {
  key: string;
  title: string | function;
  checked: boolean;
  visible: boolean;
  fixed: 'left' | 'right' | 'unFixed';
}

// 业务代码实际传入的格式（错误）
const columnChecks = ref<Array<{ prop: string; label: string; checked: boolean }>>([]);
```

业务代码完全忽略了框架的 `columnChecks` 机制，自己实现了一套，导致：
1. `TableHeaderOperation` 的 `v-model:columns` 双向绑定失效
2. 列设置的勾选/固定状态无法保存和恢复
3. 与框架的 `defaultTransform` 等机制完全脱节

**关键代码问题**（以 staff 为例）：
```typescript
// staff/index.vue - 问题代码
const columnChecks = ref<Array<{ prop: string; label: string; checked: boolean }>>([]);

// 应该使用框架返回的 columnChecks
const {
  data: tableData,
  columnChecks  // 直接使用框架返回的，不要自己定义
} = useNaivePaginatedTable({...});
```

### 1.2 其他框架能力未充分利用

| 功能 | 框架能力 | 项目实际 | 问题 |
|------|----------|----------|------|
| **列设置** | `columnChecks` from `useTable` | 自定义数组，完全绕过 | 列设置失效 |
| **图片上传** | `NUpload` 基础组件 | 每个页面自己写 | 代码重复，维护成本高 |
| **下拉选择** | `NSelect` | 直接使用，没问题 | ✅ 正确 |
| **数据图表** | 内置 ECharts | 未发现使用 | 需确认 |
| **表单生成** | `@sa/hooks` 的 form hook | 未使用 | 每个页面手写表单 |

### 1.3 字段不一致问题

| 问题 | 位置 | 错误写法 | 正确做法 |
|------|------|----------|----------|
| 字段名兼容 | `staff/index.vue:689` | `staff.staffId \|\| staff.id` | 统一后端字段名 |
| 服务类型字段 | `order/index.vue:691` | `serviceTypes \|\| serviceTypeNames` | 统一字段命名 |
| 照片格式 | `elder/index.vue` | 多格式兼容处理 | 统一后端返回格式 |

## 三、重构步骤

### 阶段一：修复列设置功能（优先级：P0）

#### 步骤 1.1：理解框架的列设置机制

框架的列设置流程：
```
useNaivePaginatedTable / useNaiveTable
    ↓
返回 columnChecks (TableColumnCheck[])
    ↓
传给 TableHeaderOperation (v-model:columns)
    ↓
传给 TableColumnSetting
    ↓
用户修改 -> 更新 columnChecks -> useTable 监听变化 -> 过滤 columns
```

#### 步骤 1.2：修改 staff/index.vue

**Before（错误）：**
```typescript
const { data: tableData, loading, pagination, columnChecks } = useNaivePaginatedTable({...});

// 错误：自己定义了 columnChecks
const columnChecks = ref<Array<{ prop: string; label: string; checked: boolean }>>([]);
```

**After（正确）：**
```typescript
// 直接使用框架返回的 columnChecks，不要重新定义
const {
  data: tableData,
  loading,
  pagination,
  mobilePagination,
  getData,
  getDataByPage,
  columnChecks  // 框架返回的，可直接用于 TableHeaderOperation
} = useNaivePaginatedTable<Api.Common.PaginatingQueryRecord<Api.Staff.Staff>, Api.Staff.Staff>({
  apiFn: async params => {...},
  apiParams: {...},
  transform: defaultTransform,
  columns: () => columns  // 注意：这里不需要传 getColumnChecks，框架已处理
});
```

#### 步骤 1.3：模板修改

**Before：**
```vue
<TableHeaderOperation
  v-model:columns="columnChecks"
  ...
/>
```

**After：**
```vue
<!-- 直接使用框架返回的 columnChecks -->
<TableHeaderOperation
  v-model:columns="columnChecks"
  :disabled-delete="checkedRowKeys.length === 0"
  :loading="loading"
  @add="handleAdd"
  @refresh="getData"
  @delete="handleBatchDelete"
/>
```

#### 步骤 1.4：其他模块同步修改

### 阶段二：组件复用（优先级：P1）

#### 步骤 2.1：抽取公共上传组件

当前问题：每个页面都自己写 NUpload 上传逻辑

建议：创建 `FileUploader` 组件
```vue
<!-- src/components/common/file-uploader.vue -->
<template>
  <NUpload
    v-model:file-list="fileList"
    :custom-request="handleUpload"
    :max="maxCount"
    ...
  >
    <NButton>上传文件</NButton>
  </NUpload>
</template>
```

#### 步骤 2.2：抽取公共表单组件

使用框架的 `@sa/hooks` form hook 或创建 `BaseForm` 组件

#### 步骤 2.3：PersonCard 组件优化

当前问题：PersonCard 只适用于员工，不适用于老人

建议：泛化组件或创建 `ElderCard`

### 阶段三：字段规范（优先级：P1）

#### 步骤 3.1：制定字段命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| ID字段 | 使用具体名称，不用泛化的 `id` | `staffId`, `orderId`, `elderId` |
| 时间字段 | 使用 `createTime/updateTime` | `createTime`, `updateTime` |
| 状态字段 | 使用 `status`，值用枚举 | `status: 'PENDING' \| 'CONFIRMED'` |
| 列表字段 | 统一单复数 | `serviceTypes`（数组）, `serviceType`（单个） |

#### 步骤 3.2：统一 API 层字段映射

在 API service 层做统一转换，而不是在每个页面兼容

## 四、规范文档

### 4.1 列表页面规范

```typescript
// ✅ 正确：直接使用框架返回的 columnChecks
const {
  data: tableData,
  loading,
  pagination,
  columnChecks,  // 直接使用，不要重新定义
  getData
} = useNaivePaginatedTable<Response, ApiData>({
  apiFn: async params => fetchList(params),
  transform: defaultTransform,
  columns: () => columns  // columns 必须有 key 属性
});

// ❌ 错误：自己定义 columnChecks
const columnChecks = ref<Array<{ prop: string; label: string; checked: boolean }>>([]);
```

### 4.2 列定义规范

```typescript
// ✅ 正确：columns 必须有 key
const columns: DataTableColumns<Api.Staff.Staff> = [
  { type: 'selection' },
  { key: 'staffName', title: '姓名', width: 120 },
  { key: 'phone', title: '电话', width: 120 },
  ...
];

// ❌ 错误：没有 key
const columns = [
  { title: '姓名', width: 120 },  // 缺少 key
  ...
];
```

### 4.3 字段映射规范

```typescript
// ✅ 正确：在 API 层统一映射
export function fetchGetStaffList(params) {
  return request({
    url: '/api/staff',
    params: {
      ...params,
      // 统一字段名
    }
  });
}

// ❌ 错误：在页面层兼容多种格式

```

## 二、修复顺序

### 批次1: 根因修复（必须先执行）

#### ✅ 修复1.1: ProviderMapper.xml 字段映射错误
**文件**: `src/main/resources/mapper/provider/ProviderMapper.xml`
**问题**: `status` 字段被映射到 `auditStatus`，而 `status` 属性没有映射
**状态**: ✅ 已完成
**修改**:
```xml
<!-- 修改前 -->
<result column="status" property="auditStatus"/>

<!-- 修改后 -->
<result column="audit_status" property="auditStatus"/>
<result column="status" property="status"/>
```

#### ✅ 修复1.2: ServiceLog Entity 缺少字段
**文件**: `src/main/java/com/elderlycare/entity/servicelog/ServiceLog.java`
**问题**: Entity 缺少 `deleted` 和 `auditStatus` 字段
**状态**: ✅ 已完成
**修改**: 添加了 `auditStatus` 和 `deleted` 字段

#### ✅ 修复1.3: Appointment 表缺少 order_id/order_no 字段
**文件**: `sql/init_missing_tables.sql`

**状态**: ✅ 已完成

#### ✅ 修复1.4: QualityCheckMapper.xml 为空

## 五、后续建议

### 立即执行（数据库DDL）
需要在生产数据库执行以下 SQL：
```sql
-- 1. appointment表添加字段
ALTER TABLE appointment
ADD COLUMN order_id VARCHAR(64) DEFAULT NULL COMMENT '关联订单ID',
ADD COLUMN order_no VARCHAR(64) DEFAULT NULL COMMENT '关联订单号';

-- 2. service_log表确认deleted字段存在（如果不存在）
ALTER TABLE service_log ADD COLUMN deleted TINYINT(1) DEFAULT 0 COMMENT '逻辑删除';

-- 3. service_log表确认audit_status字段存在（如果不存在）
ALTER TABLE service_log ADD COLUMN audit_status VARCHAR(20) DEFAULT 'PENDING' COMMENT '审核状态';
```

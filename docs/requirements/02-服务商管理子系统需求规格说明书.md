# 服务商管理子系统需求规格说明书

## 1. 概述

### 1.1 子系统简介
服务商管理子系统是智慧居家养老服务管理平台的核心组成部分，负责服务商的全生命周期管理，包括注册、认证、审核、考核、退出等环节。

### 1.2 业务目标
- 规范服务商准入机制，确保服务商具备相应资质
- 实现服务商信息的数字化管理，提高管理效率
- 建立服务商绩效考核体系，促进服务质量提升
- 保障服务商退出机制，维护服务市场秩序

## 2. 用户角色与权限

| 角色 | 权限说明 |
|------|----------|
| 市级管理员 | 全市服务商信息查看、全局服务商分类管理、服务商准入标准制定 |
| 区县管理员 | 本辖区服务商注册审核、服务商信息管理、服务商绩效考核 |
| 街道乡镇管理员 | 查看辖区内服务商信息、协调服务商资源 |
| 社区管理员 | 查看辖区内服务商信息、服务资源调配 |
| 服务商管理员 | 本企业信息维护、人员管理、订单管理、结算管理 |

## 3. 功能模块

### 3.1 服务商注册

#### 3.1.1 功能描述
提供服务商在线注册功能，支持服务商自主填写企业基本信息，提交相关资质材料。

#### 3.1.2 功能要求
- 支持服务商自主注册账号
- 支持填写企业基本信息（企业名称、统一社会信用代码、法定代表人、注册地址、经营范围、联系人、联系电话等）
- 支持上传企业资质材料（营业执照、组织机构代码证、税务登记证、养老服务机构许可证、卫生许可证、食品安全许可证等）
- 支持填写服务能力信息（服务类型、服务区域、人员规模、服务设施等）
- 支持填写财务信息（银行账户、开户行、税务登记号等）
- 支持填写服务承诺书、安全责任书
- 注册信息提交后进入待审核状态

#### 3.1.3 数据字段
```json
{
  "providerId": "服务商ID",
  "providerName": "企业名称",
  "creditCode": "统一社会信用代码",
  "legalPerson": "法定代表人",
  "legalPersonPhone": "法定代表人电话",
  "registerAddress": "注册地址",
  "businessAddress": "经营地址",
  "businessScope": "经营范围",
  "contactPerson": "联系人",
  "contactPhone": "联系电话",
  "serviceTypes": ["服务类型列表"],
  "serviceArea": "服务区域",
  "staffSize": "人员规模",
  "facilityInfo": "服务设施信息",
  "bankAccount": "银行账号",
  "bankName": "开户银行",
  "taxNumber": "税务登记号",
  "attachments": ["资质材料附件列表"],
  "commitmentLetter": "服务承诺书",
  "safetyResponsibility": "安全责任书",
  "status": "状态（待审核、审核中、已通过、已拒绝）",
  "createTime": "注册时间"
}
```

#### 3.1.4 业务规则
- 统一社会信用代码必须与工商系统核验一致
- 营业执照必须在有效期内
- 养老服务机构许可证为必填项
- 服务区域必须在注册地所在的区县范围内
- 联系电话必须真实有效

### 3.2 服务商认证审核

#### 3.2.1 功能描述
区县管理员对注册的服务商进行资质审核，确认其具备相应的服务能力。

#### 3.2.2 功能要求
- 支持查看服务商注册信息
- 支持查看上传的资质材料
- 支持与工商系统核验企业信息
- 支持通过/拒绝审核
- 支持填写审核意见
- 支持退回补充材料
- 支持批量审核
- 支持审核流程记录

#### 3.2.3 审核流程
```
提交注册 → 待审核 → 审核中 → 审核通过 / 审核拒绝 / 补充材料 → 最终通过 / 最终拒绝
```

#### 3.2.4 审核标准
| 审核项 | 审核内容 | 审核标准 |
|--------|----------|----------|
| 企业资质 | 营业执照、许可证 | 证件齐全、真实有效、经营范围涵盖养老服务 |
| 人员资质 | 养老护理员证、健康管理师证 | 主要服务人员具备相应资质 |
| 设施条件 | 办公场所、服务设施 | 符合养老服务标准 |
| 财务状况 | 银行账户、税务登记 | 财务信息完整有效 |

### 3.3 服务商信息管理

#### 3.3.1 功能描述
服务商可自主维护企业基本信息，管理员可查看和维护服务商信息。

#### 3.3.2 功能要求
- 服务商基本信息维护（企业名称、地址、联系方式等）
- 服务商资质材料更新（证件到期提醒、证件续期上传）
- 服务商服务能力更新（服务类型、服务区域、人员规模等）
- 服务商信息变更审批
- 服务商信息变更历史记录

#### 3.3.3 字段说明
- **基本信息**: 企业名称、统一社会信用代码、法定代表人、注册地址、经营地址、联系人、联系电话、邮箱等
- **服务信息**: 服务类型、服务区域、服务定价、服务时间等
- **资质信息**: 各类证书、许可证的有效期管理
- **变更记录**: 记录所有信息变更的历史

### 3.4 服务商服务范围管理

#### 3.4.1 功能描述
管理服务商提供的服务类型和服务区域。

#### 3.4.2 功能要求

- 服务区域配置（精确到街道/社区）
- 服务定价配置（政府指导价、市场调节价）
- 服务时间配置（服务时段、服务时长）
- 服务区域与服务类型关联管理
- 服务定价与补贴标准关联

### 3.6 服务商绩效考核

#### 3.6.1 功能描述
建立服务商绩效考核体系，定期对服务商服务质量进行评估。

#### 3.6.2 考核指标
| 考核维度 | 考核指标 | 权重 |
|----------|----------|------|
| 服务质量 | 客户满意度评分 | 30% |
| 服务效率 | 订单按时完成率 | 20% |
| 服务规范 | 服务达标率 | 20% |
| 投诉处理 | 投诉处理及时率 | 15% |
| 合规经营 | 合规性检查 | 15% |

#### 3.6.3 功能要求
- 自动计算考核得分
- 支持手动调整考核分数
- 生成考核报告
- 考核结果排名
- 考核结果公示
- 考核结果与服务商信用等级关联

#### 3.6.4 考核周期
- 月度考核：每月进行一次
- 季度考核：每季度进行一次综合考核
- 年度考核：每年进行一次全面考核

### 3.7 服务商信用管理

#### 3.7.1 功能描述
建立服务商信用评价体系，根据考核结果和投诉情况评定信用等级。

#### 3.7.2 信用等级
| 等级 | 得分范围 | 说明 |
|------|----------|------|
| AAA | 90-100分 | 优秀服务商 |
| AA | 80-89分 | 良好服务商 |
| A | 70-79分 | 合格服务商 |
| B | 60-69分 | 需改进服务商 |
| C | 0-59分 | 不合格服务商 |

#### 3.7.3 功能要求
- 自动计算信用得分
- 信用等级评定
- 信用等级动态更新
- 信用等级展示
- 信用等级与政策支持关联

### 3.8 服务商退出管理

#### 3.8.1 功能描述
规范服务商退出流程，确保服务商退出后服务不受影响。

#### 3.8.2 退出条件
- 服务商主动申请退出
- 服务商信用等级为C
- 服务商严重违规经营
- 服务商资质证书失效
- 服务商破产清算

#### 3.8.3 退出流程
```
服务商申请退出 / 管理员启动退出 → 停止新单派发 → 处理未完成订单 → 资金结算 → 服务商信息归档 → 完成退出
```

#### 3.8.4 功能要求
- 服务商主动申请退出
- 管理员强制退出
- 退出原因记录
- 未完成订单处理
- 资金清算
- 服务商信息归档
- 退出记录查询

### 3.9 服务商信息查询统计

#### 3.9.1 功能描述
提供多维度的服务商信息查询和统计分析功能。

#### 3.9.2 查询功能
- 按服务商名称查询
- 按服务类型查询
- 按服务区域查询
- 按信用等级查询
- 按状态查询
- 组合查询

#### 3.9.3 统计功能
- 服务商数量统计（总数、新增、退出）
- 服务商类型统计
- 服务商区域分布统计
- 服务商信用等级统计
- 服务商服务量统计
- 服务商投诉统计

## 4. 数据模型

### 4.1 服务商信息表（t_provider）
```sql
CREATE TABLE t_provider (
    provider_id VARCHAR(32) PRIMARY KEY COMMENT '服务商ID',
    provider_name VARCHAR(200) NOT NULL COMMENT '企业名称',
    credit_code VARCHAR(18) UNIQUE NOT NULL COMMENT '统一社会信用代码',
    legal_person VARCHAR(50) COMMENT '法定代表人',
    legal_person_phone VARCHAR(20) COMMENT '法定代表人电话',
    register_address VARCHAR(500) COMMENT '注册地址',
    business_address VARCHAR(500) COMMENT '经营地址',
    contact_person VARCHAR(50) COMMENT '联系人',
    contact_phone VARCHAR(20) COMMENT '联系电话',
    email VARCHAR(100) COMMENT '邮箱',
    service_area VARCHAR(500) COMMENT '服务区域',
    staff_size INT COMMENT '人员规模',
    facility_info TEXT COMMENT '服务设施信息',
    bank_account VARCHAR(50) COMMENT '银行账号',
    bank_name VARCHAR(100) COMMENT '开户银行',
    tax_number VARCHAR(50) COMMENT '税务登记号',
    credit_level VARCHAR(10) COMMENT '信用等级',
    credit_score DECIMAL(5,2) COMMENT '信用得分',
    status VARCHAR(20) COMMENT '状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
);
```

### 4.2 服务商资质表（t_provider_qualification）
```sql
CREATE TABLE t_provider_qualification (
    id VARCHAR(32) PRIMARY KEY,
    provider_id VARCHAR(32) NOT NULL COMMENT '服务商ID',
    qualification_type VARCHAR(50) COMMENT '资质类型',
    qualification_name VARCHAR(200) COMMENT '资质名称',
    qualification_number VARCHAR(100) COMMENT '资质编号',
    issue_date DATE COMMENT '发证日期',
    expiry_date DATE COMMENT '有效期至',
    attachment_url VARCHAR(500) COMMENT '附件地址',
    status VARCHAR(20) COMMENT '状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (provider_id) REFERENCES t_provider(provider_id)
);
```

### 4.3 服务商服务类型表（t_provider_service_type）
```sql
CREATE TABLE t_provider_service_type (
    id VARCHAR(32) PRIMARY KEY,
    provider_id VARCHAR(32) NOT NULL,
    service_type_code VARCHAR(20) NOT NULL,
    service_type_name VARCHAR(100) NOT NULL,
    service_price DECIMAL(10,2) COMMENT '服务价格',
    subsidy_price DECIMAL(10,2) COMMENT '补贴价格',
    status VARCHAR(20),
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (provider_id) REFERENCES t_provider(provider_id)
);
```

### 4.4 服务商考核表（t_provider_assessment）
```sql
CREATE TABLE t_provider_assessment (
    id VARCHAR(32) PRIMARY KEY,
    provider_id VARCHAR(32) NOT NULL,
    assessment_type VARCHAR(20) COMMENT '考核类型（月度/季度/年度）',
    assessment_date DATE COMMENT '考核日期',
    satisfaction_score DECIMAL(5,2) COMMENT '满意度得分',
    efficiency_score DECIMAL(5,2) COMMENT '效率得分',
    quality_score DECIMAL(5,2) COMMENT '质量得分',
    complaint_score DECIMAL(5,2) COMMENT '投诉得分',
    compliance_score DECIMAL(5,2) COMMENT '合规得分',
    total_score DECIMAL(5,2) COMMENT '总分',
    credit_level VARCHAR(10) COMMENT '信用等级',
    assessor_id VARCHAR(32) COMMENT '考核人ID',
    assessment_content TEXT COMMENT '考核内容',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (provider_id) REFERENCES t_provider(provider_id)
);
```

## 5. 接口设计

### 5.1 服务商注册接口
```
POST /api/provider/register
```

**请求参数**:
```json
{
  "providerName": "企业名称",
  "creditCode": "统一社会信用代码",
  "legalPerson": "法定代表人",
  "registerAddress": "注册地址",
  "contactPerson": "联系人",
  "contactPhone": "联系电话",
  "serviceTypes": ["服务类型列表"],
  "serviceArea": "服务区域",
  "attachments": ["资质材料"]
}
```

**响应参数**:
```json
{
  "code": 200,

  "data": {
    "providerId": "服务商ID",
    "status": "待审核"

```

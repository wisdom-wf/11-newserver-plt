# 订单管理子系统需求规格说明书

## 1. 概述

### 1.1 子系统简介
订单管理子系统是智慧居家养老服务管理平台的核心业务系统，负责订单的全生命周期管理，包括订单创建、派单、服务执行、评价、结算等环节。

### 1.2 业务目标
- 实现订单全流程数字化管理
- 提高订单处理效率，缩短服务响应时间
- 确保服务质量，提升客户满意度
- 实现订单信息的透明化和可追溯性

## 2. 用户角色与权限

| 角色 | 权限说明 |
|------|----------|
| 社区管理员 | 订单创建、派单、订单跟踪、订单评价 |
| 服务商管理员 | 订单接收、派单给服务人员、订单管理 |
| 服务人员 | 订单接收、执行服务、上传服务记录、签退 |
| 老人/家属 | 订单查看、服务评价、投诉建议 |

## 3. 功能模块

### 3.1 订单创建

#### 3.1.1 功能描述
支持社区管理员或老人/家属创建服务订单。

#### 3.1.2 功能要求
- 支持老人信息自动填充
- 支持服务类型选择
- 支持服务时间选择
- 支持服务地址确认
- 支持特殊需求备注
- 支持紧急订单标记
- 支持订单模板（常用服务快速下单）
- 支持订单预览

#### 3.1.3 订单信息字段
```json
{
  "orderId": "订单ID",
  "elderId": "老人ID",
  "elderName": "老人姓名",
  "elderPhone": "老人联系电话",
  "serviceType": "服务类型",
  "serviceName": "服务名称",
  "serviceDate": "服务日期",
  "serviceTime": "服务时间",
  "serviceDuration": "预计服务时长",
  "serviceAddress": "服务地址",
  "specialRequirements": "特殊要求",
  "orderType": "订单类型（普通/紧急）",
  "orderSource": "订单来源（社区/老人/家属）",
  "subsidyType": "补贴类型（政府补贴/自费/混合）",
  "estimatedPrice": "预计价格",
  "subsidyAmount": "补贴金额",
  "selfPayAmount": "自付金额",
  "status": "订单状态"
}
```

#### 3.1.4 业务规则
- 服务时间必须在服务商营业时间内
- 服务地址必须在服务商服务区域内
- 紧急订单需在2小时内派单
- 政府补贴需验证老人补贴资格

### 3.2 订单派单

#### 3.2.1 功能描述
将订单分派给合适的服务商或服务人员。

#### 3.2.2 派单模式
| 派单模式 | 说明 | 适用场景 |
|----------|------|----------|
| 手动派单 | 管理员手动选择服务商/人员 | 特殊需求订单、紧急订单 |
| 自动派单 | 系统根据规则自动匹配 | 普通订单、批量订单 |
| 抢单模式 | 多个服务商/人员竞争接单 | 服务人员充足区域 |

#### 3.2.3 派单规则
- 服务区域匹配
- 服务类型匹配
- 服务人员排班匹配
- 服务人员资质匹配
- 服务人员空闲时间匹配
- 服务人员距离优先（就近派单）
- 服务商信用等级优先

#### 3.2.4 功能要求
- 支持查看可派单服务商/人员列表
- 支持查看服务商/人员信息
- 支持查看服务商/人员评分
- 支持查看服务商/人员距离
- 支持批量派单
- 支持派单记录查询

#### 3.2.5 派单流程
```
待派单 → 选择服务商/人员 → 发送派单 → 服务商/人员确认 → 派单成功 / 拒单 → 重新派单
```

### 3.3 订单接收

#### 3.3.1 功能描述
服务商/服务人员接收派单或主动接单。

#### 3.3.2 功能要求
- 支持查看订单详情
- 支持接受订单
- 支持拒绝订单（需填写拒绝理由）
- 支持订单提醒（短信、APP推送）
- 支持订单超时自动取消
- 支持接单超时自动转派

#### 3.3.3 接单超时规则
| 订单类型 | 接单时限 | 超时处理 |
|----------|----------|----------|
| 紧急订单 | 10分钟 | 自动转派 |
| 普通订单 | 30分钟 | 自动转派 |

### 3.4 订单执行

#### 3.4.1 功能描述
服务人员执行服务，记录服务过程。

#### 3.4.2 执行流程
```
出发 → 到达签到 → 开始服务 → 服务过程中 → 服务结束签退 → 上传服务记录
```

#### 3.4.3 功能要求
- 支持出发提醒
- 支持导航到服务地址
- 支持位置打卡（GPS定位）
- 支持上传服务照片
- 支持上传服务视频
- 支持填写服务日志
- 支持异常情况上报
- 支持联系老人/家属

#### 3.4.4 签到规则
- 签到位置与老人住址距离<500米
- 签到时间在预定时间±15分钟内
- 签到时需上传现场照片

#### 3.4.5 签退规则
- 签退位置与老人住址距离<500米
- 服务时长≥30分钟
- 签退时需上传服务完成照片

#### 3.4.6 服务记录字段
```json
{
  "serviceRecordId": "服务记录ID",
  "orderId": "订单ID",
  "staffId": "服务人员ID",
  "serviceDate": "服务日期",
  "checkInTime": "签到时间",
  "checkOutTime": "签退时间",
  "serviceDuration": "实际服务时长",
  "checkInLocation": "签到位置（经纬度）",
  "checkOutLocation": "签退位置（经纬度）",
  "servicePhotos": ["服务照片"],
  "serviceVideo": "服务视频",
  "serviceLog": "服务日志",
  "abnormalSituation": "异常情况",
  "elderSignature": "老人签名",
  "serviceStatus": "服务状态"
}
```

### 3.5 订单状态管理

#### 3.5.1 订单状态流转
```
待派单 → 已派单 → 已接单 → 服务中 → 已完成 → 已评价 → 已结算
                    ↓
                 已取消 / 已拒绝
```

#### 3.5.2 状态说明
| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待派单 | 订单创建完成，等待派单 | 派单、取消 |
| 已派单 | 已派给服务商/人员 | 接单、拒单 |
| 已接单 | 服务商/人员已接单 | 出发、取消 |
| 服务中 | 服务正在进行中 | 签退 |
| 已完成 | 服务已结束 | 评价 |
| 已评价 | 已完成评价 | 结算 |
| 已结算 | 资金已结算完成 | 无 |
| 已取消 | 订单已取消 | 查看详情 |
| 已拒绝 | 服务商/人员拒绝接单 | 重新派单 |

#### 3.5.3 功能要求
- 自动状态流转
- 支持手动调整状态
- 状态变更日志记录
- 状态变更通知

### 3.6 订单取消

#### 3.6.1 功能描述
支持订单的取消操作。

#### 3.6.2 取消场景
- 老人/家属主动取消
- 服务商/人员无法服务
- 社区管理员取消
- 系统自动取消（超时无人接单）

#### 3.6.3 取消规则
| 取消时间 | 是否收费 | 说明 |
|----------|----------|------|
| 服务前2小时以上 | 免费 | 可全额退款 |
| 服务前1-2小时 | 收取20% | 部分退款 |
| 服务前1小时内 | 收取50% | 部分退款 |
| 服务开始后 | 收取100% | 不退款 |

#### 3.6.4 功能要求
- 支持取消订单
- 支持取消原因填写
- 支持取消审批（特殊订单）
- 支持取消记录查询

### 3.7 订单评价

#### 3.7.1 功能描述
老人/家属对服务进行评价。

#### 3.7.2 评价维度
| 评价维度 | 权重 | 评价内容 |
|----------|------|----------|
| 服务态度 | 30% | 服务人员态度是否友好、耐心 |
| 服务质量 | 30% | 服务是否专业、到位 |
| 服务效率 | 20% | 是否准时、高效 |
| 服务环境 | 10% | 服务过程是否整洁、卫生 |
| 总体满意度 | 10% | 整体满意度 |

#### 3.7.3 功能要求
- 支持5星评分
- 支持文字评价
- 支持上传评价照片
- 支持匿名评价
- 支持补充评价
- 支持评价修改（24小时内）

#### 3.7.4 评价记录字段

#### 3.7.5 评价规则
- 订单完成后7天内可评价
- 超过7天未评价默认4星
- 评价与服务商/人员评分关联

### 3.8 订单结算

#### 3.8.1 功能描述
计算订单金额，进行资金结算。

#### 3.8.2 结算方式
| 结算方式 | 说明 | 适用场景 |
|----------|------|----------|
| 在线支付 | 通过平台支付 | 自费订单 |
| 政府结算 | 政府补贴直接结算 | 政府补贴订单 |
| 混合支付 | 政府补贴+自费 | 混合订单 |
| 月结 | 每月统一结算 | 固定服务订单 |

#### 3.8.3 价格计算规则
```
实际价格 = 服务时长 × 服务单价
结算金额 = 实际价格 - 补贴金额
自付金额 = 结算金额
```

#### 3.8.4 功能要求
- 自动计算订单金额
- 支持政府补贴自动计算
- 支持生成结算单
- 支持在线支付
- 支持发票开具
- 支持结算记录查询

#### 3.8.5 结算记录字段

## 4. 数据模型

### 4.1 订单信息表（t_order）
```sql
CREATE TABLE t_order (
    order_id VARCHAR(32) PRIMARY KEY COMMENT '订单ID',
    order_no VARCHAR(50) UNIQUE NOT NULL COMMENT '订单编号',
    elder_id VARCHAR(32) NOT NULL COMMENT '老人ID',
    elder_name VARCHAR(50) COMMENT '老人姓名',
    elder_phone VARCHAR(20) COMMENT '老人电话',
    service_type_code VARCHAR(20) NOT NULL COMMENT '服务类型编码',
    service_type_name VARCHAR(100) COMMENT '服务类型名称',
    service_date DATE COMMENT '服务日期',
    service_time VARCHAR(10) COMMENT '服务时间',
    service_duration INT COMMENT '预计服务时长（分钟）',
    service_address VARCHAR(500) COMMENT '服务地址',
    special_requirements TEXT COMMENT '特殊要求',
    order_type VARCHAR(20) COMMENT '订单类型',
    order_source VARCHAR(20) COMMENT '订单来源',
    subsidy_type VARCHAR(20) COMMENT '补贴类型',
    estimated_price DECIMAL(10,2) COMMENT '预计价格',
    subsidy_amount DECIMAL(10,2) COMMENT '补贴金额',
    self_pay_amount DECIMAL(10,2) COMMENT '自付金额',
    status VARCHAR(20) COMMENT '订单状态',
    provider_id VARCHAR(32) COMMENT '服务商ID',
    staff_id VARCHAR(32) COMMENT '服务人员ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_elder (elder_id),
    INDEX idx_status (status),
    INDEX idx_date (service_date),
    INDEX idx_provider (provider_id),
    INDEX idx_staff (staff_id)
);
```

### 4.2 订单派单记录表（t_order_dispatch）
```sql
CREATE TABLE t_order_dispatch (
    dispatch_id VARCHAR(32) PRIMARY KEY,
    order_id VARCHAR(32) NOT NULL COMMENT '订单ID',
    provider_id VARCHAR(32) NOT NULL COMMENT '服务商ID',
    staff_id VARCHAR(32) COMMENT '服务人员ID',
    dispatch_time DATETIME COMMENT '派单时间',
    dispatcher_id VARCHAR(32) COMMENT '派单人ID',
    dispatch_type VARCHAR(20) COMMENT '派单类型',
    dispatch_status VARCHAR(20) COMMENT '派单状态',
    receive_time DATETIME COMMENT '接单时间',
    reject_reason VARCHAR(500) COMMENT '拒绝理由',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES t_order(order_id)
);
```

### 4.3 服务记录表（t_service_record）
```sql
CREATE TABLE t_service_record (
    service_record_id VARCHAR(32) PRIMARY KEY,
    order_id VARCHAR(32) NOT NULL,
    staff_id VARCHAR(32) NOT NULL,
    check_in_time DATETIME COMMENT '签到时间',
    check_out_time DATETIME COMMENT '签退时间',
    service_duration INT COMMENT '实际服务时长',
    check_in_location VARCHAR(500) COMMENT '签到位置',
    check_out_location VARCHAR(500) COMMENT '签退位置',
    service_photos TEXT COMMENT '服务照片',
    service_video VARCHAR(500) COMMENT '服务视频',
    service_log TEXT COMMENT '服务日志',
    abnormal_situation TEXT COMMENT '异常情况',
    elder_signature VARCHAR(500) COMMENT '老人签名',
    service_status VARCHAR(20) COMMENT '服务状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES t_order(order_id),
    FOREIGN KEY (staff_id) REFERENCES t_staff(staff_id)
);
```

### 4.4 订单评价表（t_order_evaluation）
```sql
CREATE TABLE t_order_evaluation (
    evaluation_id VARCHAR(32) PRIMARY KEY,
    order_id VARCHAR(32) NOT NULL,
    staff_id VARCHAR(32) NOT NULL,
    provider_id VARCHAR(32) NOT NULL,
    attitude_score INT COMMENT '态度得分',
    quality_score INT COMMENT '质量得分',
    efficiency_score INT COMMENT '效率得分',
    environment_score INT COMMENT '环境得分',
    overall_score INT COMMENT '总体得分',
    average_score DECIMAL(5,2) COMMENT '平均得分',
    evaluation_content TEXT COMMENT '评价内容',
    evaluation_photos TEXT COMMENT '评价照片',
    is_anonymous TINYINT COMMENT '是否匿名',
    evaluation_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES t_order(order_id)
);
```

### 4.5 订单结算表（t_order_settlement）
```sql
CREATE TABLE t_order_settlement (
    settlement_id VARCHAR(32) PRIMARY KEY,
    order_id VARCHAR(32) NOT NULL,
    provider_id VARCHAR(32) NOT NULL,
    staff_id VARCHAR(32) NOT NULL,
    service_duration INT COMMENT '实际服务时长',
    unit_price DECIMAL(10,2) COMMENT '服务单价',
    total_amount DECIMAL(10,2) COMMENT '总金额',
    subsidy_amount DECIMAL(10,2) COMMENT '补贴金额',
    self_pay_amount DECIMAL(10,2) COMMENT '自付金额',
    payment_method VARCHAR(20) COMMENT '支付方式',
    payment_status VARCHAR(20) COMMENT '支付状态',
    settlement_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    invoice_status VARCHAR(20) COMMENT '发票状态',
    FOREIGN KEY (order_id) REFERENCES t_order(order_id)
);
```

### 4.6 订单投诉表（t_order_complaint）
```sql
CREATE TABLE t_order_complaint (
    complaint_id VARCHAR(32) PRIMARY KEY,
    order_id VARCHAR(32) NOT NULL,
    complainant_id VARCHAR(32) COMMENT '投诉人ID',
    complainant_type VARCHAR(20) COMMENT '投诉人类型',
    complaint_type VARCHAR(50) COMMENT '投诉类型',
    complaint_content TEXT COMMENT '投诉内容',
    complaint_evidence TEXT COMMENT '投诉证据',
    complaint_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    handler_id VARCHAR(32) COMMENT '处理人ID',
    handle_result TEXT COMMENT '处理结果',
    handle_time DATETIME COMMENT '处理时间',
    complaint_status VARCHAR(20) COMMENT '投诉状态',
    FOREIGN KEY (order_id) REFERENCES t_order(order_id)
);
```

**响应参数**:

{
  "code": 200,
  "message": "订单创建成功",

  }

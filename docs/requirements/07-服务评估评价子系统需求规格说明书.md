# 服务评估评价子系统需求规格说明书

## 1. 概述

### 1.1 子系统简介
服务评估评价子系统是智慧居家养老服务管理平台的质量管理核心模块，负责对服务质量进行全方位评价，收集客户反馈，处理服务争议，为服务质量提升和绩效考核提供数据支撑。

### 1.2 业务目标
- 建立完善的服务评价体系
- 实现服务质量的客观评估
- 及时收集客户反馈意见
- 规范处理服务争议
- 为服务商和服务人员绩效考核提供依据

## 2. 用户角色与权限

| 角色 | 权限说明 |
|------|----------|
| 市级管理员 | 全市服务质量查看、评价统计分析、质量监管 |
| 区县管理员 | 本辖区服务质量查看、评价统计分析、质量监管 |
| 街道乡镇管理员 | 本辖区服务质量查看、评价统计分析 |
| 社区管理员 | 社区服务质量查看、评价收集、投诉处理 |
| 服务商管理员 | 本企业服务评价查看、评价回复、质量改进 |
| 服务人员 | 个人服务评价查看、评价回复、服务质量改进 |
| 老人/家属 | 服务评价、投诉建议、评价查看 |

## 3. 功能模块

### 3.1 服务评价收集

#### 3.1.1 功能描述
收集老人/家属对服务的评价。

#### 3.1.2 评价触发时机
- 订单服务完成后
- 定期回访时
- 主动评价时

#### 3.1.3 功能要求
- 支持订单完成后自动推送评价
- 支持主动发起评价
- 支持定期回访评价
- 支持匿名评价
- 支持评价时间限制（订单完成后7天内）
- 支持评价修改（24小时内）
- 支持评价补填（超过7天未评价自动4星）

#### 3.1.4 评价方式
| 评价方式 | 说明 | 适用场景 |
|----------|------|----------|
| 在线评价 | 通过APP/小程序评价 | 大部分订单 |
| 电话回访 | 专人电话回访 | 重点老人、特殊订单 |
| 上门回访 | 社区人员上门回访 | 高龄老人、失能老人 |
| 短信评价 | 通过短信回复评价 | 不使用智能手机的老人 |

#### 3.1.5 评价维度
| 评价维度 | 权重 | 评分标准 |
|----------|------|----------|
| 服务态度 | 30% | 服务人员是否友好、耐心、尊重 |
| 服务质量 | 30% | 服务是否专业、到位、符合要求 |
| 服务效率 | 20% | 是否准时、高效 |
| 服务环境 | 10% | 服务过程是否整洁、卫生、安全 |
| 总体满意度 | 10% | 整体服务满意度 |

### 3.2 服务质量评分

#### 3.2.1 功能描述
对服务商和服务人员的服务质量进行综合评分。

#### 3.2.2 评分周期
| 评分周期 | 评分范围 | 更新频率 |
|----------|----------|----------|
| 日评分 | 当日服务评价 | 每日更新 |
| 周评分 | 当周服务评价 | 每周更新 |
| 月评分 | 当月服务评价 | 每月更新 |
| 季度评分 | 当季度服务评价 | 每季度更新 |
| 年度评分 | 当年服务评价 | 每年更新 |

#### 3.2.3 评分计算公式
```
服务质量得分 = (态度评分 × 30% + 质量评分 × 30% + 效率评分 × 20% + 环境评分 × 10% + 总体评分 × 10%)
```

#### 3.2.4 功能要求
- 自动计算服务商服务质量得分
- 自动计算服务人员服务质量得分
- 支持手动调整评分
- 支持评分历史查看
- 支持评分排名
- 支持评分趋势分析

### 3.3 评价管理

#### 3.3.1 功能描述
管理服务评价的审核、回复、修改等。

#### 3.3.2 功能要求
- 支持评价审核（恶意评价、虚假评价）
- 支持评价隐藏（隐私信息）
- 支持评价修改（24小时内）
- 支持评价删除（特殊情况）
- 支持评价回复（服务商/服务人员）
- 支持评价举报
- 支持评价置顶（优秀评价）

#### 3.3.3 评价审核
| 审核类型 | 审核内容 | 处理方式 |
|----------|----------|----------|
| 恶意评价 | 含辱骂、诽谤等内容 | 隐藏或删除 |
| 虚假评价 | 非本人评价、刷单评价 | 删除 |
| 隐私评价 | 包含个人敏感信息 | 隐蔽处理 |
| 重复评价 | 同一订单多次评价 | 保留最新一次 |

#### 3.3.4 评价回复
- 服务商/服务人员可对评价进行回复
- 回复内容包括感谢、解释、改进措施等
- 回复时间限制：收到评价后7天内
- 回复字数限制：最多500字

#### 3.4.3 功能要求

- 支持反馈分类管理

### 3.5 服务争议处理

#### 3.5.1 功能描述
处理老人/家属与服务商/服务人员之间的服务争议。

#### 3.5.2 争议类型
| 争议类型 | 常见情况 | 处理方式 |
|----------|----------|----------|
| 服务质量争议 | 服务未达标准 | 调查核实，协商解决 |
| 服务费用争议 | 费用计算争议 | 核实计算，多退少补 |
| 服务时间争议 | 服务时间不足 | 补充服务或退款 |
| 服务态度争议 | 服务人员态度问题 | 道歉、换人、培训 |
| 其他争议 | 其他类型争议 | 根据具体情况处理 |

#### 3.5.3 争议处理流程
```
争议申请 → 争议登记 → 调查核实 → 协商调解 → 达成协议 / 调解失败 → 执行协议 / 仲裁或诉讼 → 争议结案
```

#### 3.5.4 功能要求
- 支持争议在线申请
- 支持争议登记管理
- 支持争议调查记录
- 支持争议调解记录
- 支持争议协议管理
- 支持争议跟踪查询
- 支持争议统计分析

## 4. 数据模型

### 4.1 服务评价表（t_service_evaluation）
```sql
CREATE TABLE t_service_evaluation (
    evaluation_id VARCHAR(32) PRIMARY KEY COMMENT '评价ID',
    order_id VARCHAR(32) NOT NULL COMMENT '订单ID',
    staff_id VARCHAR(32) NOT NULL COMMENT '服务人员ID',
    provider_id VARCHAR(32) NOT NULL COMMENT '服务商ID',
    elder_id VARCHAR(32) COMMENT '老人ID',
    evaluation_type VARCHAR(20) COMMENT '评价类型',
    attitude_score INT COMMENT '态度得分',
    quality_score INT COMMENT '质量得分',
    efficiency_score INT COMMENT '效率得分',
    environment_score INT COMMENT '环境得分',
    overall_score INT COMMENT '总体得分',
    average_score DECIMAL(5,2) COMMENT '平均得分',
    evaluation_content TEXT COMMENT '评价内容',
    evaluation_tags VARCHAR(500) COMMENT '评价标签',
    evaluation_photos TEXT COMMENT '评价照片',
    is_anonymous TINYINT COMMENT '是否匿名',
    evaluation_time DATETIME COMMENT '评价时间',
    reply_status VARCHAR(20) COMMENT '回复状态',
    reply_content TEXT COMMENT '回复内容',
    reply_time DATETIME COMMENT '回复时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES t_order(order_id),
    INDEX idx_staff (staff_id),
    INDEX idx_provider (provider_id),
    INDEX idx_evaluation_time (evaluation_time)
);
```

### 4.2 服务商评分表（t_provider_score）
```sql
CREATE TABLE t_provider_score (
    provider_score_id VARCHAR(32) PRIMARY KEY,

    start_date DATE COMMENT '开始日期',
    end_date DATE COMMENT '结束日期',
    total_orders INT COMMENT '总订单数',
    evaluated_orders INT COMMENT '已评价订单数',
    evaluation_rate DECIMAL(5,2) COMMENT '评价率',
    attitude_score DECIMAL(5,2) COMMENT '态度平均分',
    quality_score DECIMAL(5,2) COMMENT '质量平均分',
    efficiency_score DECIMAL(5,2) COMMENT '效率平均分',
    environment_score DECIMAL(5,2) COMMENT '环境平均分',

    average_score DECIMAL(5,2) COMMENT '综合得分',
    rank_num INT COMMENT '排名',
    rank_change INT COMMENT '排名变化',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_provider_period (provider_id, score_period, start_date, end_date)
);
```

### 4.3 服务人员评分表（t_staff_score）

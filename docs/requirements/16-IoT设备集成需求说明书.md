# 智慧养老服务管理平台 - IoT设备集成需求规格说明书

> 文档版本: v1.0
> 更新日期: 2026-04-04

## 1. IoT集成概述

### 1.1 设计目标

通过IoT物联网技术，实现对老人居家安全的实时监护，构建"感知-分析-预警-处置"全链路闭环。

### 1.2 设备矩阵

| 设备类型 | 品牌参考 | 采集数据 | 应用场景 |
|----------|----------|----------|----------|
| 智能床垫 | 惠达/美的 | 睡眠质量、心率、在床/离床 | 夜间安全监护 |
| 智能血压计 | 欧姆龙 | 血压、脉搏 | 健康监测 |
| 智能血糖仪 | 三诺 | 血糖值 | 慢病管理 |
| 智能手环 | 华为/小米 | 心率、步数、睡眠、GPS | 全天候监护 |
| 燃气报警器 | 海曼 | 燃气浓度 | 居家安全 |
| 烟雾报警器 | 海康 | 烟雾浓度 | 火灾预防 |
| 智能门磁 | 门窗闭合 | 开关门记录 | 活动轨迹 |
| GPS定位器 | 老人手表 | 位置数据 | 防走失 |
| 紧急呼叫器 | 一键呼叫 | 求助信号 | 紧急救援 |
| 智能药盒 | 服药提醒 | 用药记录 | 慢病管理 |

## 2. 数据采集规范

### 2.1 数据格式

```json
{
  "deviceId": "DEVICE_001",
  "deviceType": "smart_bed",
  "userId": "ELDER_001",
  "timestamp": "2026-04-04T10:30:00Z",
  "data": {
    "heartRate": 72,
    "breathingRate": 16,
    "sleepStatus": "sleeping",
    "inBed": true,
    "bedLeavingCount": 0
  },
  "location": {
    "province": "浙江省",
    "city": "杭州市",
    "district": "西湖区",
    "address": "文三路XXX号"
  }
}
```

### 2.2 采集频率

| 设备类型 | 采集频率 | 上报频率 | 说明 |
|----------|----------|----------|------|
| 智能床垫 | 1次/分钟 | 5分钟批量 | 睡眠数据批量上报 |
| 智能血压计 | 按需 | 实时 | 用户测量时上报 |
| 智能血糖仪 | 按需 | 实时 | 用户测量时上报 |
| 智能手环 | 1次/5分钟 | 实时 | 持续监测 |
| 燃气报警器 | 1次/秒 | 实时 | 阈值触发立即上报 |
| 烟雾报警器 | 1次/秒 | 实时 | 阈值触发立即上报 |
| 智能门磁 | 状态变化时 | 实时 | 开关门时上报 |
| GPS定位器 | 1次/分钟 | 实时 | 移动时上报 |
| 紧急呼叫器 | 按需 | 实时 | 按下时上报 |
| 智能药盒 | 状态变化 | 实时 | 服药/未服药时上报 |

### 2.3 数据质量

| 要求 | 说明 |
|------|------|
| 完整性 | 数据缺失率<1% |
| 时效性 | 数据延迟<10秒 |
| 准确性 | 设备校准后误差<5% |
| 可用性 | 服务可用性>99.9% |

## 3. 设备对接协议

### 3.1 协议选择

| 协议 | 适用场景 | 说明 |
|------|----------|------|
| MQTT | 设备直连 | 轻量级，适合低功耗设备 |
| HTTP(s) | 设备/网关 | 通用协议，易集成 |
| CoAP | NB-IoT设备 | 适合物联网卡设备 |
| TCP私有协议 | 特殊设备 | 厂商自定义协议 |

### 3.2 设备入网流程

```
设备上电
    ↓
连接配置（WiFi/蜂窝网络）
    ↓
连接IoT网关/平台
    ↓
设备认证（设备证书）
    ↓
注册设备信息
    ↓
开始数据上报
```

### 3.3 安全要求

| 要求 | 实现方式 |
|------|----------|
| 设备认证 | 设备证书 + Token |
| 数据加密 | TLS 1.3 |
| 固件升级 | 签名验证 + 差分升级 |
| 访问控制 | 设备ID白名单 |

## 4. 数据处理

### 4.1 流处理架构

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  设备数据   │───▶│   MQTT      │───▶│   Flink     │
│  (JSON)    │    │   Broker    │    │  流处理     │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                              │
                    ┌──────────────────────────┼──────────────┐
                    │                          │              │
                    ▼                          ▼              ▼
           ┌─────────────┐           ┌─────────────┐ ┌─────────────┐
           │  实时告警   │           │   时序库    │ │   数据仓库   │
           │  (Redis)   │           │ (InfluxDB)  │ │ (ClickHouse) │
           └─────────────┘           └─────────────┘ └─────────────┘
```

### 4.2 异常检测规则

#### 4.2.1 智能床垫

| 规则 | 条件 | 阈值 | 动作 |
|------|------|------|------|
| 离床超时 | 夜间离床超过 | 30分钟 | 电话确认 |
| 心率异常 | 持续异常 | >150或<50 | 预警通知 |
| 无活动 | 长时间无活动 | 6小时 | 确认安全 |
| 睡眠质量差 | 睡眠效率 | <60% | 健康提醒 |

#### 4.2.2 燃气报警器

| 规则 | 条件 | 阈值 | 动作 |
|------|------|------|------|
| 燃气泄漏 | 浓度超标 | >10%LEL | 立即报警+切断 |
| 持续升高 | 浓度持续上升 | 5分钟 | 紧急撤离 |

#### 4.2.3 智能门磁

| 规则 | 条件 | 阈值 | 动作 |
|------|------|------|------|
| 门长时间未开 | 设定时间内 | 24小时 | 电话确认 |
| 异常开门 | 非正常时间 | 22:00-06:00 | 确认安全 |

#### 4.2.4 GPS定位器

| 规则 | 条件 | 阈值 | 动作 |
|------|------|------|------|
| 电子围栏 | 离开安全区域 | 立即 | 预警通知 |
| 电量低 | 电量低于 | 20% | 充电提醒 |
| 静止超时 | 设备静止 | 1小时 | 确认安全 |

## 5. 预警处置

### 5.1 处置流程

```
异常检测
    ↓
预警分级（红/橙/黄/蓝）
    ↓
预警通知（多渠道）
    ↓
确认处置
    ↓
结果反馈
    ↓
记录归档
```

### 5.2 预警分级

| 等级 | 场景 | 通知方式 | 时限 |
|------|------|----------|------|
| 红色 | 燃气泄漏/老人跌倒/紧急呼叫 | 电话+短信+APP+120 | 立即 |
| 橙色 | 心率异常/长时间离床 | 短信+APP | 5分钟 |
| 黄色 | 门磁异常/用药提醒 | APP+短信 | 30分钟 |
| 蓝色 | 电量低/设备离线 | APP推送 | 24小时 |

### 5.3 通知策略

| 通知对象 | 红色 | 橙色 | 黄色 | 蓝色 |
|----------|------|------|------|------|
| 老人本人 | ✓ | - | - | - |
| 家属1 | ✓ | ✓ | ✓ | - |
| 家属2 | ✓ | ✓ | - | - |
| 社区管家 | ✓ | ✓ | ✓ | - |
| 120急救 | ✓ | - | - | - |
| 物业 | ✓ | - | - | - |

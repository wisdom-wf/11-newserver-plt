# 服务人员管理子系统需求规格说明书

## 1. 概述

### 1.1 子系统简介
服务人员管理子系统负责管理养老服务人员的全生命周期，包括人员档案、资质认证、排班调度、绩效考核、离职管理等环节。

### 1.2 业务目标
- 建立完善的服务人员档案管理体系
- 确保服务人员具备相应资质，提供合格服务
- 实现服务人员的科学调度，提高服务效率
- 建立绩效考核体系，促进服务质量提升

## 2. 用户角色与权限

| 角色 | 权限说明 |
|------|----------|
| 服务商管理员 | 服务人员档案管理、资质认证、排班调度、绩效考核、离职管理 |
| 服务人员 | 个人信息维护、排班查看、任务接收、工作记录上传 |

## 3. 功能模块

### 3.1 服务人员档案管理

#### 3.1.1 功能描述
建立和维护服务人员的完整档案信息。

#### 3.1.2 功能要求
- 支持服务人员基本信息录入
- 支持个人照片上传
- 支持紧急联系人信息管理
- 支持工作经历记录
- 支持教育背景记录
- 支持技能特长记录
- 支持档案信息修改
- 支持档案信息导入导出

  "gender": "性别（男/女）",

  "age": "年龄",
  "idCard": "身份证号",
  "phone": "联系电话",
  "email": "邮箱",
  "address": "家庭住址",
  "emergencyContact": "紧急联系人姓名",

#### 3.1.4 业务规则

- 联系电话必须真实有效
- 年龄范围：18-65周岁
- 紧急联系人信息必须填写

#### 3.2.2 功能要求

- 支持资质证书扫描件上传
- 支持资质证书有效期管理

- 支持资质证书续期
- 支持资质证书过期处理

| 健康管理师证 | 健康管理师职业资格证书 | 人力资源和社会保障部 |

| 心理咨询师证 | 心理咨询师职业资格证书 | 人力资源和社会保障部 |

| 营养师证 | 公共营养师职业资格证书 | 人力资源和社会保障部 |

  "staffId": "人员ID",

  "qualificationNumber": "证书编号",
  "issueDate": "发证日期",

  "issueOrganization": "发证机构",

#### 3.2.5 到期提醒机制
- 距离到期30天：系统提醒服务商
- 距离到期15天：系统再次提醒服务商
- 距离到期7天：系统强制提醒服务商
- 已过期：禁止该人员接单

- 服务时长：最短30分钟，最长4小时

### 3.5 服务人员绩效考核管理

#### 3.5.1 功能描述
建立服务人员绩效考核体系，定期评估服务人员服务质量。

#### 3.5.2 考核指标
| 考核维度 | 考核指标 | 权重 |
|----------|----------|------|
| 服务态度 | 客户满意度 | 25% |
| 服务质量 | 服务达标率 | 25% |
| 工作效率 | 订单完成率 | 20% |
| 出勤情况 | 出勤率 | 15% |
| 规范执行 | 操作规范执行率 | 15% |

#### 3.5.3 考核等级
| 等级 | 得分范围 | 奖惩措施 |
|------|----------|----------|
| 优秀 | 90-100分 | 奖金+晋升机会 |
| 良好 | 80-89分 | 奖金 |
| 合格 | 70-79分 | 正常待遇 |
| 待改进 | 60-69分 | 培训+警告 |
| 不合格 | 0-59分 | 停止派单+培训或辞退 |

#### 3.5.4 功能要求
- 自动计算考核得分
- 支持手动调整
- 生成考核报告
- 考核结果通知
- 考核结果与薪资关联

## 4. 数据模型

### 4.1 服务人员信息表（t_staff）
```sql
CREATE TABLE t_staff (
    staff_id VARCHAR(32) PRIMARY KEY COMMENT '人员ID',
    provider_id VARCHAR(32) NOT NULL COMMENT '所属服务商ID',
    staff_name VARCHAR(50) NOT NULL COMMENT '姓名',
    gender VARCHAR(10) COMMENT '性别',
    birth_date DATE COMMENT '出生日期',
    id_card VARCHAR(18) UNIQUE NOT NULL COMMENT '身份证号',
    phone VARCHAR(20) COMMENT '联系电话',
    email VARCHAR(100) COMMENT '邮箱',
    address VARCHAR(500) COMMENT '家庭住址',
    emergency_contact VARCHAR(50) COMMENT '紧急联系人',
    emergency_phone VARCHAR(20) COMMENT '紧急联系人电话',
    education VARCHAR(20) COMMENT '学历',
    work_experience TEXT COMMENT '工作经历',
    skills TEXT COMMENT '技能特长',
    photo_url VARCHAR(500) COMMENT '个人照片',
    status VARCHAR(20) COMMENT '状态',
    hire_date DATE COMMENT '入职日期',
    work_status VARCHAR(20) COMMENT '工作状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_phone (phone),
    INDEX idx_id_card (id_card),
    INDEX idx_provider (provider_id)
);
```

### 4.2 服务人员资质表（t_staff_qualification）
```sql
CREATE TABLE t_staff_qualification (
    qualification_id VARCHAR(32) PRIMARY KEY,
    staff_id VARCHAR(32) NOT NULL COMMENT '人员ID',
    qualification_type VARCHAR(50) COMMENT '资质类型',
    qualification_name VARCHAR(200) COMMENT '证书名称',
    qualification_number VARCHAR(100) COMMENT '证书编号',
    issue_date DATE COMMENT '发证日期',
    expiry_date DATE COMMENT '有效期至',
    issue_organization VARCHAR(200) COMMENT '发证机构',
    qualification_level VARCHAR(20) COMMENT '资质等级',
    attachment_url VARCHAR(500) COMMENT '证书扫描件',
    status VARCHAR(20) COMMENT '状态',
    audit_status VARCHAR(20) COMMENT '审核状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (staff_id) REFERENCES t_staff(staff_id),
    INDEX idx_expiry (expiry_date)
);
```

### 4.3 服务人员排班表（t_staff_schedule）
```sql
CREATE TABLE t_staff_schedule (
    schedule_id VARCHAR(32) PRIMARY KEY,
    staff_id VARCHAR(32) NOT NULL,
    schedule_date DATE NOT NULL COMMENT '排班日期',
    shift_type VARCHAR(20) COMMENT '班次类型',
    start_time VARCHAR(10) COMMENT '开始时间',
    end_time VARCHAR(10) COMMENT '结束时间',
    work_area VARCHAR(200) COMMENT '服务区域',
    service_types TEXT COMMENT '服务类型',
    schedule_status VARCHAR(20) COMMENT '排班状态',
    remark TEXT COMMENT '备注',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (staff_id) REFERENCES t_staff(staff_id),
    INDEX idx_date (schedule_date)
);
```

### 4.4 服务人员工作记录表（t_staff_work_record）
```sql
CREATE TABLE t_staff_work_record (
    work_record_id VARCHAR(32) PRIMARY KEY,
    staff_id VARCHAR(32) NOT NULL,
    order_id VARCHAR(32) COMMENT '订单ID',
    elder_id VARCHAR(32) COMMENT '老人ID',
    service_date DATE COMMENT '服务日期',
    check_in_time DATETIME COMMENT '签到时间',
    check_out_time DATETIME COMMENT '签退时间',
    service_duration INT COMMENT '服务时长（分钟）',
    check_in_location VARCHAR(500) COMMENT '签到位置',
    check_out_location VARCHAR(500) COMMENT '签退位置',
    service_photos TEXT COMMENT '服务照片',
    service_video VARCHAR(500) COMMENT '服务视频',
    service_log TEXT COMMENT '服务日志',
    elder_feedback TEXT COMMENT '老人反馈',
    work_status VARCHAR(20) COMMENT '工作状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (staff_id) REFERENCES t_staff(staff_id),
    INDEX idx_date (service_date)
);
```

### 4.5 服务人员考核表（t_staff_assessment）
```sql
CREATE TABLE t_staff_assessment (
    assessment_id VARCHAR(32) PRIMARY KEY,
    staff_id VARCHAR(32) NOT NULL,
    assessment_type VARCHAR(20) COMMENT '考核类型',
    assessment_date DATE COMMENT '考核日期',
    satisfaction_score DECIMAL(5,2) COMMENT '满意度得分',
    quality_score DECIMAL(5,2) COMMENT '质量得分',
    efficiency_score DECIMAL(5,2) COMMENT '效率得分',
    attendance_score DECIMAL(5,2) COMMENT '出勤得分',
    compliance_score DECIMAL(5,2) COMMENT '规范得分',
    total_score DECIMAL(5,2) COMMENT '总分',
    assessment_level VARCHAR(20) COMMENT '考核等级',
    assessor_id VARCHAR(32) COMMENT '考核人ID',
    assessment_content TEXT COMMENT '考核内容',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (staff_id) REFERENCES t_staff(staff_id)
);
```

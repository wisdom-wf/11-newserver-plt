# 财务结算管理子系统需求规格说明书

## 1. 概述

### 1.1 子系统简介
财务结算管理子系统是智慧居家养老服务管理平台的核心财务系统，负责服务定价、补贴计算、支付结算、财务报表等功能，实现资金管理的规范化、透明化和自动化。

### 1.2 业务目标
- 实现服务定价的科学管理
- 确保政府补贴的精准发放
- 实现支付结算的便捷高效
- 提供准确的财务数据支持决策

## 2. 用户角色与权限

| 角色 | 权限说明 |
|------|----------|
| 市级管理员 | 全市资金查看、定价策略制定、统计分析 |
| 区县管理员 | 本辖区资金监管、补贴审核、财务对账 |
| 街道乡镇管理员 | 本辖区资金使用查看、报表查询 |
| 服务商管理员 | 服务收入查看、结算单查看、对账 |
| 财务人员 | 财务对账、报表生成、资金结算 |

### 3.3 支付结算管理

#### 3.3.1 功能描述
管理订单的支付和结算流程。

#### 3.3.2 支付方式
| 支付方式 | 说明 | 手续费 |
|----------|------|--------|
| 在线支付 | 微信支付、支付宝等 | 0.6% |
| 银联支付 | 银联卡支付 | 0.5% |
| 政府结算 | 政府补贴直接结算 | 免费 |
| 线下支付 | 现金、POS机等 | 按实际情况 |
| 月结 | 定期统一结算 | 免费 |

#### 3.3.3 结算流程
```
订单完成 → 计算金额 → 生成结算单 → 确认结算 → 支付款项 → 资金到账 → 开具发票
```

#### 3.3.4 功能要求

- 支持多种支付方式
- 支持在线支付集成
- 支持支付结果回调
- 支持退款处理
- 支持发票开具

#### 3.5.1 功能描述

| 退款类型 | 退款方式 | 处理时限 |

| 部分退款 | 原路退回 | 3个工作日 |
| 补贴退回 | 退回补贴账户 | 实时 |

- 政府补贴部分退回补贴账户
- 自费部分原路退回

## 4. 数据模型

### 4.1 服务定价表（t_service_price）
```sql
CREATE TABLE t_service_price (
    price_id VARCHAR(32) PRIMARY KEY COMMENT '定价ID',
    service_type_code VARCHAR(20) NOT NULL COMMENT '服务类型编码',
    service_type_name VARCHAR(100) COMMENT '服务类型名称',
    price_unit VARCHAR(20) COMMENT '计价单位',
    standard_price DECIMAL(10,2) COMMENT '标准价格',
    government_price DECIMAL(10,2) COMMENT '政府指导价',
    subsidy_price DECIMAL(10,2) COMMENT '补贴价格',
    area VARCHAR(100) COMMENT '适用区域',
    effective_date DATE COMMENT '生效日期',
    expiry_date DATE COMMENT '失效日期',
    status VARCHAR(20) COMMENT '状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_service_type (service_type_code)
);
```

### 4.2 补贴标准表（t_subsidy_standard）
```sql
CREATE TABLE t_subsidy_standard (
    subsidy_standard_id VARCHAR(32) PRIMARY KEY COMMENT '补贴标准ID',
    subsidy_type VARCHAR(50) COMMENT '补贴类型',
    subsidy_name VARCHAR(100) COMMENT '补贴名称',
    subsidy_amount DECIMAL(10,2) COMMENT '补贴金额',
    subsidy_unit VARCHAR(20) COMMENT '补贴单位',
    qualification_conditions TEXT COMMENT '资格条件',
    effective_date DATE COMMENT '生效日期',
    expiry_date DATE COMMENT '失效日期',
    priority INT COMMENT '优先级',
    status VARCHAR(20) COMMENT '状态',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4.3 结算单表（t_settlement）
```sql
CREATE TABLE t_settlement (
    settlement_id VARCHAR(32) PRIMARY KEY,
    settlement_no VARCHAR(50) UNIQUE COMMENT '结算编号',
    order_id VARCHAR(32) NOT NULL COMMENT '订单ID',
    provider_id VARCHAR(32) COMMENT '服务商ID',
    staff_id VARCHAR(32) COMMENT '服务人员ID',
    elder_id VARCHAR(32) COMMENT '老人ID',
    service_date DATE COMMENT '服务日期',
    service_duration INT COMMENT '实际服务时长',
    unit_price DECIMAL(10,2) COMMENT '服务单价',
    total_amount DECIMAL(10,2) COMMENT '总金额',
    subsidy_amount DECIMAL(10,2) COMMENT '补贴金额',
    self_pay_amount DECIMAL(10,2) COMMENT '自付金额',
    payment_method VARCHAR(20) COMMENT '支付方式',
    payment_status VARCHAR(20) COMMENT '支付状态',
    payment_time DATETIME COMMENT '支付时间',
    transaction_id VARCHAR(100) COMMENT '交易流水号',
    settlement_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '结算时间',
    invoice_status VARCHAR(20) COMMENT '发票状态',
    invoice_no VARCHAR(50) COMMENT '发票号码',
    FOREIGN KEY (order_id) REFERENCES t_order(order_id),
    INDEX idx_settlement_no (settlement_no),
    INDEX idx_provider (provider_id),
    INDEX idx_elder (elder_id)
);
```

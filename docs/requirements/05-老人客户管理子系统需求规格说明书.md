# 老人/客户管理子系统需求规格说明书

## 1. 概述

### 1.1 子系统简介
老人/客户管理子系统是智慧居家养老服务管理平台的核心基础模块，负责建立和维护服务对象的完整档案，包括基本信息、健康状况、服务需求、政府补贴等。

### 1.2 业务目标
- 建立完整的老人档案管理体系
- 实现老人信息的数字化管理
- 准确掌握老人服务需求
- 规范政府补贴管理
- 提升养老服务精准度

## 2. 用户角色与权限

| 角色 | 权限说明 |
|------|----------|
| 市级管理员 | 全市老人信息查看、数据统计分析 |
| 区县管理员 | 本辖区老人信息管理、补贴资格审核 |
| 街道乡镇管理员 | 本辖区老人信息管理、服务需求收集 |
| 社区管理员 | 社区老人信息录入、更新、服务派单 |
| 老人/家属 | 个人信息查看、服务申请、投诉建议 |

## 3. 功能模块

### 3.1 老人档案管理

#### 3.1.1 功能描述
建立和维护老人的基本信息档案。

#### 3.1.2 功能要求
- 支持老人基本信息录入
- 支持老人照片上传
- 支持家庭信息管理
- 支持联系人信息管理
- 支持居住信息管理
- 支持档案信息修改
- 支持档案信息导入导出
- 支持档案归档（老人去世、迁出）

- 居住地址必须在服务范围内

### 3.4 政府补贴管理

#### 3.4.1 功能描述
管理老人的政府补贴资格和补贴使用情况。

| 经济困难老人补贴 | 固定金额 | 低保、特困老人 |
| 重度残疾老人补贴 | 固定金额 | 重度残疾老人 |
| 其他补贴 | 根据政策 | 其他符合条件的老人 |

## 4. 数据模型

### 4.1 老人基本信息表（t_elder）
```sql
CREATE TABLE t_elder (
    elder_id VARCHAR(32) PRIMARY KEY COMMENT '老人ID',
    elder_no VARCHAR(50) UNIQUE COMMENT '老人编号',
    elder_name VARCHAR(50) NOT NULL COMMENT '姓名',
    gender VARCHAR(10) COMMENT '性别',
    birth_date DATE COMMENT '出生日期',
    age INT COMMENT '年龄',
    id_card VARCHAR(18) UNIQUE NOT NULL COMMENT '身份证号',
    phone VARCHAR(20) COMMENT '联系电话',
    ethnicity VARCHAR(20) COMMENT '民族',
    education VARCHAR(20) COMMENT '学历',
    marital_status VARCHAR(20) COMMENT '婚姻状况',
    political_status VARCHAR(20) COMMENT '政治面貌',
    native_place VARCHAR(200) COMMENT '籍贯',
    photo_url VARCHAR(500) COMMENT '个人照片',
    status VARCHAR(20) COMMENT '状态',
    register_date DATE COMMENT '建档日期',
    address VARCHAR(500) COMMENT '居住地址',
    area VARCHAR(100) COMMENT '所属区域',
    community VARCHAR(100) COMMENT '所属社区',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_name (elder_name),
    INDEX idx_id_card (id_card),
    INDEX idx_area (area),
    INDEX idx_community (community)
);
```

### 4.2 老人家庭信息表（t_elder_family）
```sql
CREATE TABLE t_elder_family (
    family_id VARCHAR(32) PRIMARY KEY,
    elder_id VARCHAR(32) NOT NULL COMMENT '老人ID',
    family_type VARCHAR(20) COMMENT '家庭关系类型',
    family_name VARCHAR(50) COMMENT '家属姓名',
    family_phone VARCHAR(20) COMMENT '家属电话',
    family_address VARCHAR(500) COMMENT '家属地址',
    is_caretaker TINYINT COMMENT '是否主要照顾人',
    is_emergency_contact TINYINT COMMENT '是否紧急联系人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (elder_id) REFERENCES t_elder(elder_id)
);
```

### 4.3 老人健康信息表（t_elder_health）
```sql
CREATE TABLE t_elder_health (
    health_id VARCHAR(32) PRIMARY KEY,
    elder_id VARCHAR(32) NOT NULL COMMENT '老人ID',
    blood_type VARCHAR(10) COMMENT '血型',
    height DECIMAL(5,2) COMMENT '身高',
    weight DECIMAL(5,2) COMMENT '体重',
    medical_history TEXT COMMENT '病史记录',
    medication_list TEXT COMMENT '用药清单',
    allergy_history TEXT COMMENT '过敏史',
    chronic_diseases TEXT COMMENT '慢性病',
    disabilities TEXT COMMENT '残疾情况',
    self_care_ability VARCHAR(20) COMMENT '自理能力',
    care_level VARCHAR(20) COMMENT '照护等级',
    last_checkup_date DATE COMMENT '上次体检日期',
    checkup_report_url VARCHAR(500) COMMENT '体检报告',
    health_status VARCHAR(50) COMMENT '健康状况',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (elder_id) REFERENCES t_elder(elder_id),
    UNIQUE KEY uk_elder (elder_id)
);
```

### 4.4 老人服务需求表（t_elder_demand）
```sql
CREATE TABLE t_elder_demand (

### 4.7 老人评估表（t_elder_assessment）

    assessment_id VARCHAR(32) PRIMARY KEY,

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (elder_id) REFERENCES t_elder(elder_id)

```

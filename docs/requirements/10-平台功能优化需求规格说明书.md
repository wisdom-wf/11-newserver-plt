# 智慧养老平台功能优化需求规格说明书

**文档版本**: v1.0
**创建日期**: 2026-05-24
**编制人**: 产品经理
**文档目的**: 将视频宣传内容转化为可执行的系统开发需求

---

## 一、背景与目标

### 1.1 需求背景

基于平台"智慧养老平台功能介绍系列视频"的宣传内容，提炼出四大核心功能模块的系统优化需求。这些需求源于平台对服务质量的坚守、对服务人员权益的保障、以及对客户反馈的重视。

### 1.2 优化目标

| 目标 | 说明 |
|------|------|
| **规范化** | 建立标准化的服务商准入机制和服务人员管理体系 |
| **透明化** | 实现服务质量、客户评价的全流程透明展示 |
| **智能化** | 基于数据驱动，实现智能派单、问题预警、改进建议 |
| **激励化** | 建立服务人员激励体系，形成良性竞争 |

## 二、服务商入驻管理模块

### 2.1 需求场景

"谁在为我们的老人提供服务？我们如何放心？"

平台需要建立严格的服务商入驻审核机制，确保只有通过层层审核的服务商才能为老人提供服务。

### 2.2 功能需求

#### 2.2.1 资质认证管理

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 营业执照管理 | 服务商入驻时必须上传营业执照，支持证照有效期管理，到期前提醒续期 | P0 |
| 健康证管理 | 要求服务人员上传健康证，建立健康档案，支持到期提醒 | P0 |
| 服务合同管理 | 支持电子合同签署与管理，合同模板管理，到期自动提醒 | P1 |
| 证照核验 | 对接工商系统实现营业执照自动核验，健康证真伪查询 | P2 |

#### 2.2.2 服务类型配置

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 服务范围定义 | 服务商入驻时选择可提供的服务类型（生活照料、康复护理、助餐服务等） | P0 |
| 能力匹配 | 根据服务商的专业能力，精准匹配合适的服务项目 | P0 |
| 专人专岗 | 支持按服务类型配置服务人员，实现专人专岗 | P1 |

#### 2.2.3 信用评价体系

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 服务评分 | 每次服务完成后，客户可对服务商进行评分（1-5星） | P0 |
| 动态排名 | 根据评分、投诉率、服务量等指标，实时更新服务商排名 | P1 |
| 淘汰机制 | 评分持续低于阈值或投诉率超标的服务商，自动触发退出流程 | P0 |
| 信用档案 | 记录服务商历史评分、投诉记录、整改记录，形成完整信用档案 | P1 |

#### 2.2.4 持续监管

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 定期审核 | 支持配置定期审核周期，到期自动提醒服务商更新资质 | P1 |
| 实时监督 | 平台可实时查看服务商的服务记录和服务质量数据 | P1 |

### 2.3 数据要求

- 服务商证照影像文件存储（支持 JPG、PNG、PDF）
- 证照有效期字段（start_date, end_date, auto_remind_days）
- 信用评分计算公式：
  ```
  综合评分 = (5日平均分 × 0.3) + (30日平均分 × 0.4) + (累计平均分 × 0.3) - 投诉扣分
  ```

## 三、服务人员管理模块

### 3.1 需求场景

"每一位服务人员，都在用自己的双手传递温暖。他们的辛苦，丁峰养老看在眼里。"

平台需要建立完善的服务人员管理体系，既保障服务人员权益，又确保服务质量。

### 3.2 功能需求

#### 3.2.1 人员档案管理

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 实名认证 | 服务人员注册需完成实名认证，支持身份证OCR识别 | P0 |
| 健康档案 | 建立服务人员健康档案，健康证有效期管理 | P0 |
| 专业证书 | 支持上传和管理专业技能证书，支持证书过期提醒 | P1 |
| 档案查询 | 支持按姓名、证书类型、技能等级等条件查询服务人员 | P1 |

#### 3.2.2 服务范围智能匹配

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 技能评估 | 服务人员入驻时进行技能评估，确定可服务项目范围 | P0 |
| 服务区域 | 服务人员设置可服务区域（按街道/社区划分） | P0 |
| 智能派单 | 系统优先派送周边订单给服务人员，减少奔波 | P1 |
| 距离计算 | 根据服务人员当前位置和服务地址，计算最短距离 | P2 |

#### 3.2.3 权益保障

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 工时统计 | 自动统计服务人员实际工时，支持按月查询 | P0 |
| 薪资核算 | 根据服务项目定价和服务时长，自动核算薪资 | P0 |
| 薪资明细 | 展示薪资构成：基础工资 + 绩效奖金 + 加班费 | P1 |
| 保险保障 | 平台为服务人员购买意外保险，支持保单查询 | P1 |

#### 3.2.4 激励与奖励

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 服务评分展示 | 服务人员可查看自己的服务评分和历史评价 | P0 |
| 奖金计算 | 根据评分和客户满意度，计算并发放奖金 | P1 |
| 订单奖励 | 高评分服务人员优先获得更多优质订单 | P2 |
| 荣誉墙 | 评选优秀服务人员并展示，激励先进 | P2 |

#### 3.2.5 安全措施

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 紧急联系人 | 服务人员可设置紧急联系人，紧急情况下一键呼叫 | P0 |
| 实时定位 | 服务人员可选择共享位置，平台可查看当前位置 | P1 |
| 风险预警 | 识别异常服务场景（如深夜、偏远地区），提前预警 | P2 |
| 一键求助 | 服务人员遇到困难可一键向平台求助 | P1 |

## 四、服务质检管理模块

### 4.1 需求场景

"对老人来说，每一次服务都可能影响他们的健康。服务质量，是丁峰养老坚守的底线。"

平台需要建立严格的质检管理体系，确保服务质量真实可靠。

### 4.2 功能需求

#### 4.2.1 质检标准制定

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 服务规范 | 制定各类服务的标准和操作规范文档 | P0 |
| 检查清单 | 针对每类服务制定检查清单项，支持勾选完成情况 | P0 |
| 评分标准 | 定义评分维度和各维度权重，形成统一评分标准 | P0 |

#### 4.2.2 多维质检方式

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 现场抽查 | 质控人员可发起现场抽查，记录检查结果 | P1 |
| 照片记录 | 服务人员上传服务过程照片，作为服务凭证 | P0 |
| 客户反馈 | 服务完成后自动推送满意度调查，收集客户反馈 | P0 |
| 抽检计划 | 支持配置随机抽检比例和频率，系统自动生成抽检任务 | P2 |

#### 4.2.3 问题整改流程

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 问题登记 | 发现质量问题后，在系统中登记问题详情 | P0 |
| 整改通知 | 系统自动通知服务商和服务人员限期整改 | P0 |
| 复核确认 | 整改完成后，质控人员复核确认 | P0 |
| 暂停接单 | 未整改合格的服务商/服务人员，暂停接单资格 | P1 |

#### 4.2.4 数据分析与改进

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 问题统计 | 统计高频问题类型、问题发生时间段、服务项目 | P1 |
| 原因分析 | 归类问题原因（人员能力、设备故障、流程缺陷等） | P2 |
| 改进建议 | 系统基于数据自动生成改进建议 | P2 |
| 培训建议 | 根据问题数据，推荐需要培训的服务商/人员 | P2 |
| 优秀案例 | 筛选高分服务案例，形成优秀案例库供学习 | P1 |

#### 4.2.5 优秀案例推广

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 案例入库 | 将高分服务案例标记为优秀案例入库 | P1 |
| 案例分享 | 支持案例详情查看，经验总结分享 | P1 |
| 推广通知 | 新增优秀案例时，通知相关服务商和服务人员学习 | P2 |

## 五、满意度调查模块

### 5.1 需求场景

"每一次服务结束后，我们想问：您满意吗？您的声音，丁峰养老认真倾听。"

平台需要建立完善的满意度调查系统，收集客户反馈并形成闭环。

### 5.2 功能需求

#### 5.2.1 调查时机设计

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 自动触发 | 服务完成后自动推送满意度调查问卷 | P0 |
| 推送渠道 | 支持短信、微信小程序、APP等多渠道推送 | P0 |
| 填写入口 | 客户点击链接或扫描二维码即可填写 | P0 |
| 填写期限 | 设置填写有效期，超期未填写自动提醒 | P1 |

#### 5.2.2 调查内容设计

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 服务态度 | 评价服务人员服务态度（1-5分） | P0 |
| 专业程度 | 评价服务人员专业能力（1-5分） | P0 |
| 准时性 | 评价是否准时到达（1-5分） | P0 |
| 综合评分 | 整体满意度评分（1-5星） | P0 |
| 文字反馈 | 支持客户填写文字意见或建议 | P0 |
| 图片上传 | 支持客户上传现场照片作为凭证 | P2 |

#### 5.2.3 反馈闭环机制

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 问题追踪 | 客户反馈的问题自动关联到对应订单和服务商 | P0 |
| 改进通知 | 发现问题后，平台通知服务商和服务人员改进 | P0 |
| 结果反馈 | 整改完成后，向客户反馈处理结果 | P1 |
| 闭环确认 | 客户确认问题已解决，形成完整闭环 | P2 |

#### 5.2.4 数据价值应用

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 服务优化 | 分析满意度数据，找出服务薄弱环节 | P1 |
| 人员培训 | 根据反馈数据，推荐需要培训的服务人员 | P2 |
| 平台升级 | 汇总分析数据，为平台功能优化提供依据 | P1 |
| 报表导出 | 支持导出满意度统计报表 | P1 |

#### 5.2.5 客户激励机制

| 功能点 | 需求描述 | 优先级 |
|--------|----------|--------|
| 积分奖励 | 客户参与满意度调查即可获得积分 | P1 |
| 积分规则 | 配置积分奖励规则（如首次评价得100积分） | P1 |
| 积分兑换 | 积分可兑换优惠券或礼品 | P2 |
| 感谢回馈 | 在评价完成页面展示感谢语，提升参与意愿 | P1 |

## 六、非功能需求

### 6.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面响应时间 | 操作类页面 < 2秒，查询类页面 < 3秒 |
| 并发用户数 | 支持同时在线用户数 ≥ 500 |
| 数据处理 | 满意度调查结果T+1生成统计报表 |

### 6.2 安全需求

| 需求 | 说明 |
|------|------|
| 数据加密 | 客户隐私数据（身份证号、联系方式）加密存储 |

| 操作审计 | 关键操作记录审计日志 |

### 6.3 兼容性需求

| 需求 | 说明 |
|------|------|
| 浏览器支持 | Chrome、Firefox、Safari、Edge最新版本 |
| 移动端适配 | 支持手机端查看和填写满意度调查 |
| 微信兼容 | 支持微信内置浏览器查看和填写 |

## 七、确认事项

以下事项需与业务方确认后细化：

1. **服务商准入评分阈值**: 评分低于多少分需要退出或暂停接单？
2. **服务人员保险方案**: 保险由平台统一购买，预留接口即可，服务人员增加一个是否有保险的状态字段
3. **积分兑换礼品池**: 积分可兑换的礼品范围和兑换比例，先不做细化，预留接口即可
5. **优秀案例评选周期**: 按月、按季度评选优秀服务人员，月度之星 季度之星 做成标签体现在服务人员相关的资料中

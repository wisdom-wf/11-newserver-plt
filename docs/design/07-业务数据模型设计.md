# 业务数据模型设计

> 基于预约数据示例.csv分析的完整业务数据模型

## 一、业务闭环

```
预约信息导入 → 审核确认 → 分配服务商 → 服务商接单 → 指派服务人员
    ↓                                                           ↓
作废/取消                                                     上门服务
    ↓                                                           ↓
                                                              服务日志 → 老人签字
    ↓                                                           ↓
                                                        服务质检 → 满意度评价
    ↓                                                           ↓
                                                        结算归档
```

## 三、状态流转

### 预约状态
```
PENDING → CONFIRMED → ASSIGNED
   ↓          ↓
INVALID   CANCELLED
```

### 订单状态
```
PENDING → ASSIGNED → ACCEPTED → IN_SERVICE → COMPLETED → ARCHIVED
   ↓                                      ↓
CANCELLED                              （归档）
```

### 质检结果
```
QUALIFIED
    or
NEED_RECTIFY → IN_PROGRESS → COMPLETED → VERIFIED
                                    ↓
                                 FAILED（再次整改）
```

## 四、区域表设计

5级行政区域：

```sql
CREATE TABLE t_area (
    id VARCHAR(32) PRIMARY KEY,
    area_code VARCHAR(50) UNIQUE NOT NULL,
    area_name VARCHAR(100),
    parent_id VARCHAR(32),
    level VARCHAR(20),  -- PROVINCE/CITY/DISTRICT/STREET/COMMUNITY
    province_code VARCHAR(20),
    city_code VARCHAR(20),
    district_code VARCHAR(20),
    street_code VARCHAR(20),
    community_code VARCHAR(20),
    longitude DECIMAL(10,6),
    latitude DECIMAL(10,6),
    sort_order INT,
    status VARCHAR(20) DEFAULT 'ENABLE',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 五、API接口

### 预约管理
```
GET    /api/appointments           # 列表
GET    /api/appointments/{id}     # 详情
POST   /api/appointments/import   # Excel导入
POST   /api/appointments          # 创建
PUT    /api/appointments/{id}/confirm  # 确认
PUT    /api/appointments/{id}/assign    # 分配
PUT    /api/appointments/{id}/cancel   # 取消
PUT    /api/appointments/{id}/invalidate # 作废
```

### 订单管理
```
GET    /api/orders                # 列表
GET    /api/orders/{id}          # 详情
POST   /api/orders               # 创建
POST   /api/orders/appointment/{id}  # 从预约创建
PUT    /api/orders/{id}/assign   # 分配
PUT    /api/orders/{id}/accept    # 接单
PUT    /api/orders/{id}/start    # 开始服务
PUT    /api/orders/{id}/complete  # 完成
PUT    /api/orders/{id}/cancel   # 取消
```

### 服务日志
```
GET    /api/service-logs          # 列表
GET    /api/service-logs/order/{orderId}  # 订单日志
POST   /api/service-logs          # 提交
PUT    /api/service-logs/{id}     # 更新
PUT    /api/service-logs/{id}/anomaly  # 异常
```

### 服务质检
```
GET    /api/quality-checks       # 列表
GET    /api/quality-checks/{id}  # 详情
POST   /api/quality-checks/order/{orderId}  # 创建
PUT    /api/quality-checks/{id}/rectify  # 整改
PUT    /api/quality-checks/{id}/recheck   # 复检
```

### 满意度评价
```
GET    /api/evaluations           # 列表
GET    /api/evaluations/{id}    # 详情
POST   /api/evaluations          # 创建
PUT    /api/evaluations/{id}/reply  # 回复
```

# 管理员驾驶舱（Cockpit）设计方案

## 一、设计目标

| 目标 | 说明 |
|------|------|
| 数据分级 | 每一个级别的管理员只能看到对应级别的数据 |
| 全局视角 | 超级管理员可查看全局数据 |
| 高级UI | 现代化数据可视化，支持明/暗主题 |
| 丰富指标 | 覆盖老人、服务商、订单、财务、质检等全业务 |
| 交互操作 | 支持下钻、联动、筛选、对比等 |

## 二、角色数据权限

### 2.1 权限层级

```
超级管理员（全局）
        │
        ├── 市级管理员A
        │       ├── 区县管理员A
        │       │       ├── 街道乡镇管理员A
        │       │       └── 社区管理员A
        │       └── 区县管理员B
        └── 市级管理员B
```

### 2.2 数据可见范围

| 角色 | 可见数据范围 |
|------|-------------|
| 超级管理员 | 全国/全省数据 |
| 市级管理员 | 本市数据 |
| 区县管理员 | 本区县数据 |
| 街道乡镇管理员 | 本街道/镇数据 |
| 社区管理员 | 本社区数据 |
| 服务商管理员 | 本服务商数据 |
| 服务人员 | 个人工单数据 |

## 三、驾驶舱页面结构

### 3.1 五大专题页面

| 页面 | 定位 | 核心指标 |
|------|------|----------|
| 综合驾驶舱首页 | 核心KPI总览 | 6大核心指标+图表 |
| 老人服务专题 | 老人数据分析 | 在册/新增/流失/失能/补贴 |
| 订单分析专题 | 订单全流程 | 订单量/完成率/好评率 |
| 财务分析专题 | 收支分析 | 收入/支出/净利润 |
| 服务质量专题 | 质检与满意度 | 覆盖率/合格率/满意度 |

## 四、指标体系

### 4.1 核心KPI指标（首页）

| 指标 | 说明 | 变化趋势 |
|------|------|----------|
| 老人总数 | 在册老人数量 | 较上月% |
| 服务商总数 | 认证服务商数 | 较上月% |
| 服务人员总数 | 在职服务人员 | 较上月% |
| 本月订单 | 当月订单总量 | 较上月% |
| 本月收入 | 当月收入金额 | 较上月% |
| 满意度评分 | 平均满意度 | 较上月% |

### 4.2 下钻路径

```
老人服务 → 老人总数 → 老人列表（按区域/年龄/护理等级）
订单分析 → 今日订单 → 订单列表
服务商分析 → 收入排名 → 服务商收入明细
```

## 七、API接口

### 7.1 驾驶舱首页

```
GET /api/cockpit/dashboard
```

返回：
```typescript
{
  elder: { total, newMonth, growthRate, trend },
  provider: { total, active, avgRating },
  staff: { total, active, avgScore },
  order: { today, month, completed, completionRate },
  financial: { monthIncome, monthExpense },
  quality: { checkRate, qualifiedRate, avgScore }
}
```

### 7.2 专题分析

```
GET /api/cockpit/elder-analysis
GET /api/cockpit/order-analysis
GET /api/cockpit/financial-analysis
GET /api/cockpit/quality-analysis
```

### 7.3 下钻接口

```
GET /api/cockpit/drill-down?type=area&dimension=宝塔区&level=2&metric=order
```

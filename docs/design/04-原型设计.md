# 智慧居家养老服务管理平台 - 原型设计

## 1. 原型设计概述

### 1.1 设计目标

- 提供清晰的用户界面结构和交互流程
- 确保不同角色用户获得一致的体验
- 遵循无障碍设计原则，支持老年用户操作
- 实现响应式设计，适配多种设备
- 融入AI能力，提供智能交互体验

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 一致性 | 统一的视觉风格和交互模式 |
| 简洁性 | 界面简洁明了，减少认知负担 |
| 可用性 | 符合用户操作习惯，易于学习 |
| 可访问性 | 考虑老年用户特点，字体清晰、按钮足够大 |
| 反馈性 | 操作结果及时反馈 |
| 智能化 | AI能力自然融入，提供智能辅助 |

### 1.3 响应式断点

| 设备 | 屏幕宽度 | 布局 |
|------|----------|------|
| 桌面端 | ≥1200px | 多列布局，侧边栏展开 |
| 平板端 | 768px-1199px | 两列布局，侧边栏折叠 |
| 移动端 | <768px | 单列布局，底部导航 |

### 1.4 实际UI特征

基于系统截图分析，实际UI具有以下特征：

| 特征 | 说明 |
|------|------|
| 主色调 | 深蓝色渐变 (#003375 → #001852) |
| 数据卡片 | 深色背景 + 白色数字 + 浅色图标 |
| 地图集成 | 高德地图 (AMap) 全屏展示 |
| 图表 | ECharts 风格图表 |
| 布局 | 左侧菜单 + 右侧内容区 |
| 卡片 | 白色背景 + 阴影 + 圆角 |

### 1.5 现代化UI趋势

| 趋势 | 应用场景 |
|------|----------|
| 玻璃拟态 | 数据卡片、弹窗背景 |
| 深色模式 | 系统设置、监控大屏 |
| 微动画 | 加载状态、过渡效果 |
| AI对话 | 智能客服入口 |
| 实时数据 | 数字跳动动画 |

## 7. 老年用户适配设计

### 7.1 字体放大

- 默认使用较大字体
- 提供字体切换功能（正常/大/超大）
- 标题18px以上，正文16px以上

### 7.2 颜色对比度

- 文字与背景对比度符合WCAG 2.1 AA标准（4.5:1）
- 重要信息使用高对比度配色
- 避免纯灰色文字

### 7.3 操作友好

- 按钮足够大（高度≥36px）
- 点击区域足够大（≥48px）
- 操作反馈明确（加载状态、成功/失败提示）
- 撤销功能（允许撤销操作）

### 7.4 简化操作

- 减少操作步骤
- 提供操作引导
- 常用操作置顶
- 智能默认值

### 7.5 辅助功能

- 支持键盘操作
- 表单输入即时验证
- 错误信息清晰明确
- 提供帮助提示

# 智慧居家养老服务管理平台 - 接口设计

## 1. 接口设计概述

### 1.1 接口规范

本系统采用RESTful API规范，所有接口遵循以下统一规范：

| 规范类型 | 规范内容 |
|----------|----------|
| 通信协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 接口版本 | /api/v1 |
| 认证方式 | JWT Bearer Token |

### 1.2 基础URL

| 环境 | 基础URL |
|------|----------|
| 开发环境 | http://localhost:8080/api/v1 |
| 测试环境 | https://api-test.example.com/api/v1 |
| 生产环境 | https://api.example.com/api/v1 |

### 1.3 通用请求头

| 请求头 | 说明 | 示例 |
|--------|------|------|
| Content-Type | application/json | application/json |
| Authorization | Bearer Token | Bearer eyJhbGciOiJIUzI1NiIs... |
| X-Request-Id | 请求唯一标识 | uuid-xxx-xxx |
| X-Tenant-Id | 租户ID | 000000 |
| X-User-Id | 用户ID | 10001 |

### 1.4 通用响应格式

**成功响应**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1712131200000,
  "requestId": "uuid-xxx-xxx"
}
```

**分页响应**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "pages": 10,
    "list": []
  },
  "timestamp": 1712131200000,
  "requestId": "uuid-xxx-xxx"
}
```

**错误响应**:
```json
{
  "code": 400,
  "message": "参数错误",
  "errors": [
    {
      "field": "phone",
      "message": "手机号格式不正确"
    }
  ],
  "timestamp": 1712131200000,
  "requestId": "uuid-xxx-xxx"
}
```

### 1.5 状态码定义

| 状态码 | 说明 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | - |
| 201 | 创建成功 | - |
| 204 | 删除成功 | 无返回body |
| 400 | 参数错误 | 检查请求参数 |
| 401 | 未授权 | 重新登录 |
| 403 | 禁止访问 | 检查权限 |
| 404 | 资源不存在 | 检查ID |
| 409 | 资源冲突 | 如重复提交 |
| 422 | 业务错误 | 查看message |
| 429 | 请求过于频繁 | 降低请求频率 |
| 500 | 服务器错误 | 联系技术支持 |

## 2. 用户认证模块 (user-service)

### 2.1 用户登录

**接口**: POST /api/v1/auth/login

**请求参数**:
```json
{
  "username": "string",    // 用户名
  "password": "string",     // 密码
  "captcha": "string",     // 验证码(可选)
```

```json
{

  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
```

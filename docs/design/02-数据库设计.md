# 智慧居家养老服务管理平台 - 数据库设计

## 1. 数据库概述

### 1.1 数据库选型

| 数据库 | 版本 | 用途 | 选型理由 |
|--------|------|------|----------|
| MySQL | 8.0 | 主数据库 | 开源免费、支持事务、生态成熟、社区活跃 |

### 1.2 数据库命名规范

| 规范类型 | 规范内容 | 示例 |
|----------|----------|------|
| 数据库名 | 业务名称+环境 | `elderly_care_dev` |
| 表名 | 模块名_业务名 | `t_provider_info` |
| 字段名 | 驼峰命名 | `providerName` |
| 索引名 | idx_字段名 | `idx_provider_name` |
| 外键名 | fk_表名_字段名 | `fk_order_provider` |

### 1.3 分库分表策略

#### 1.3.1 分库策略

| 库名 | 业务模块 | 说明 |
|------|----------|------|
| `elderly_care_user` | 用户服务 | 用户、角色、权限 |
| `elderly_care_provider` | 服务商服务 | 服务商、资质、评分 |
| `elderly_care_staff` | 人员服务 | 服务人员、资质、排班 |
| `elderly_care_elder` | 老人服务 | 老人档案、健康、需求 |
| `elderly_care_order` | 订单服务 | 订单、派单、结算 |
| `elderly_care_financial` | 财务服务 | 定价、补贴、支付 |
| `elderly_care_evaluation` | 评价服务 | 评价、反馈、争议 |
| `elderly_care_config` | 配置服务 | 字典、参数、日志 |
| `elderly_care_statistics` | 统计服务 | 统计数据、报表 |

#### 1.3.2 分表策略

**订单表按年月分表**:
```
t_order_202601, t_order_202602, ...
```

**统计数据按周期分表**:
```
t_stat_data_day, t_stat_data_month, t_stat_data_year
```

## 2. 用户服务数据库 (elderly_care_user)

### 2.1 用户表 (t_user)

```sql
CREATE TABLE t_user (
    user_id VARCHAR(32) PRIMARY KEY COMMENT '用户ID',
    username VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
    password VARCHAR(200) NOT NULL COMMENT '密码(BCrypt加密)',
    real_name VARCHAR(50) COMMENT '真实姓名',
    phone VARCHAR(20) COMMENT '手机号',
    email VARCHAR(100) COMMENT '邮箱',
    gender VARCHAR(10) COMMENT '性别',
    avatar VARCHAR(500) COMMENT '头像URL',
    user_type VARCHAR(20) COMMENT '用户类型',
    status VARCHAR(20) COMMENT '状态(正常/禁用/锁定/过期)',
    last_login_time DATETIME COMMENT '最后登录时间',
    last_login_ip VARCHAR(50) COMMENT '最后登录IP',
    login_fail_count INT DEFAULT 0 COMMENT '登录失败次数',
    lock_time DATETIME COMMENT '锁定时间',
    tenant_id VARCHAR(32) COMMENT '租户ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_username (username),
    INDEX idx_phone (phone),
    INDEX idx_tenant (tenant_id),
    INDEX idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 11. 索引优化建议

### 11.1 单列索引

为所有经常用于查询的字段创建单列索引：
- 外键字段
- 状态字段
- 时间字段
- 编号字段

### 11.2 复合索引

为多条件查询创建复合索引：

```sql
-- 订单表复合索引
CREATE INDEX idx_order_status_date ON t_order_202601(status, service_date);
CREATE INDEX idx_order_provider_staff ON t_order_202601(provider_id, staff_id);

-- 老人表复合索引
CREATE INDEX idx_elder_area_status ON t_elder(area_id, status);

-- 服务人员表复合索引
CREATE INDEX idx_staff_provider_status ON t_staff(provider_id, status, work_status);
```

### 11.3 全文索引

为需要全文搜索的字段创建全文索引：

```sql
-- 服务商表全文索引
CREATE FULLTEXT INDEX ft_provider_name ON t_provider(provider_name);

-- 老人表全文索引
CREATE FULLTEXT INDEX ft_elder_name ON t_elder(elder_name);
```

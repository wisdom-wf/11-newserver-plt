# 智慧居家养老服务管理平台 - 系统架构设计

## 1. 架构概述

### 1.1 架构原则

| 原则 | 说明 |
|------|------|
| 高可用性 | 系统可用性≥99.9%，关键服务多节点部署 |
| 可扩展性 | 支持水平扩展，应对业务增长 |
| 高性能 | 响应时间<500ms，支持高并发 |
| 安全性 | 多层安全防护，数据加密存储 |
| 可维护性 | 模块化设计，便于维护升级 |
| 开放性 | 支持第三方系统集成 |
| 智能化 | AI能力融入各层，提升效率 |

### 2.5 数据架构设计

#### 2.5.1 数据存储策略

| 数据类型 | 存储方案 | 选型理由 |
|----------|----------|----------|
| 业务数据 | MySQL主从集群 | 支持事务、数据一致性强 |
| 缓存数据 | Redis Cluster | 高性能、支持分布式 |
| 搜索数据 | Elasticsearch Cluster | 全文搜索、分析能力强 |
| 文件数据 | MinIO分布式存储 | 兼容S3、扩展性好 |
| 日志数据 | Elasticsearch + Kibana | 日志分析、可视化 |
| 向量数据 | Milvus Cluster | 相似度检索、RAG |
| 时序数据 | InfluxDB/TDengine | IoT数据存储分析 |
| 分析数据 | ClickHouse | OLAP分析 |
| AI特征 | Redis + feature store | 特征缓存复用 |

#### 2.5.2 数据分片策略

**用户数据分片**:
- 按租户ID分片：Tenant-Id (000000)
- 每个租户独立数据库或Schema

**订单数据分片**:
- 按订单ID取模分片：Order-Id % 16
- 按时间分表：按年月分表

**IoT时序数据分片**:
- 按设备ID分片
- 按时间分表：日表/月表

**统计数据分片**:
- 按统计周期分表：日表、周表、月表

### 4.1 安全分层

```
┌─────────────────────────────────────────────────────────┐
│              网络安全层                               │
│  DDoS防护、WAF防火墙、IP白名单、端口限制          │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│              接入安全层                               │
│  HTTPS加密、API签名、防重放攻击、限流控制          │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│              AI安全层                               │
│  模型可解释性、偏见检测、输入过滤、输出审核      │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│              应用安全层                               │
│  认证授权、权限控制、数据加密、SQL注入防护        │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│              数据安全层                               │
│  敏感数据加密、数据脱敏、访问审计、数据备份        │
└─────────────────────────────────────────────────────────┘
```

### 4.2 认证授权机制

#### 4.2.1 认证流程

```
用户登录
    ↓
用户名密码验证 + 活体认证（可选）
    ↓
生成JWT Token (有效时间2小时)
    ↓
生成Refresh Token (有效时间7天)
    ↓
返回Token给客户端
    ↓
后续请求携带Token
    ↓
网关验证Token有效性
    ↓
解析用户信息和权限
    ↓
转发请求到目标服务
```

#### 4.2.2 权限模型

```
角色(RBAC)模型
├── 用户(User)
│   └── 角色(Role)
│       └── 权限(Permission)
│           ├── 菜单权限
│           ├── 按钮权限
│           └── 接口权限
│
└── 数据权限(Data Scope)
    ├── 全部数据
    ├── 本部门数据
    ├── 本部门及以下数据
    └── 仅本人数据
```

# 列表列设置（Column Checks）功能文档

## 1. 核心机制

列设置功能由 **SoybeanAdmin 框架** 的 `use-table.ts` 和 `table.ts` 提供，支持：
- 列的显示/隐藏
- 列的固定位置（left/right/unFixed）
- **localStorage 持久化**（页面刷新后设置不丢失）
- 移动端适配（mobilePagination）

## 2. 涉及的关键文件

| 文件 | 职责 |
|------|------|
| `packages/hooks/src/use-table.ts` | 底层 table hook，**添加了 localStorage 持久化** |
| `src/hooks/common/table.ts` | 导出 `useNaivePaginatedTable` 和 `useTableOperate` |
| `src/components/advanced/table-header-operation.vue` | 列设置 UI 组件（勾选面板） |
| 各业务模块 views/*/index.vue | 使用 hook 的页面 |

## 3. 存储机制

### localStorage Key 生成规则

```typescript
// packages/hooks/src/use-table.ts
const storageKey = `table_column_checks_${columns.toString().slice(0, 50)}`;
```

**注意**：Key 基于列定义数组的 `toString()` 结果的前50字符生成。如果列定义不变，Key 就固定。

### 保存/加载逻辑

```typescript
function loadSavedChecks(): TableColumnCheck[] | null {
  try {
    const json = localStorage.getItem(storageKey);
    return json ? JSON.parse(json) : null;
  } catch { return null; }
}

function saveChecks(checks: TableColumnCheck[]) {
  try { localStorage.setItem(storageKey, JSON.stringify(checks)); } catch {}
}

const savedChecks = loadSavedChecks();
const columnChecks = ref(savedChecks || getColumnChecks(columns()));

watch(columnChecks, newChecks => { saveChecks(newChecks); }, { deep: true });
```

## 4. 业务模块使用规范（必须遵循）

### 4.1 标准引入模式

```typescript
import { useNaivePaginatedTable, useTableOperate, defaultTransform } from '@/hooks/common/table';
import TableHeaderOperation from '@/components/advanced/table-header-operation.vue';
```

### 4.2 变量命名规范（重要！避免变量遮蔽）

| 用途 | 变量名 | 说明 |
|------|--------|------|
| 列定义（传给框架前） | `tableColumns` | 避免与框架的 `columns` 冲突 |
| 列定义（框架处理后） | `filteredColumns` | 解构时重命名 `columns: filteredColumns` |
| 列设置状态 | `columnChecks` | v-model 绑定到 TableHeaderOperation |

```typescript
// ❌ 错误：columns 变量名遮蔽框架返回值
const { columns, ... } = useNaivePaginatedTable({ columns: () => columns ... });

// ✅ 正确：列定义用 tableColumns，框架返回的列用 filteredColumns
const tableColumns: DataTableColumns<Api.xxx.Entity> = [...];

const {
  data: tableData,
  loading,
  pagination,
  mobilePagination,
  getData,
  getDataByPage,
  columns: filteredColumns,    // 重命名！
  columnChecks                  // 列设置状态
} = useNaivePaginatedTable<...>({
  apiFn: async params => { ... },
  transform: defaultTransform,
  columns: () => tableColumns   // 传入函数返回列定义
});
```

### 4.3 useTableOperate 必须引入（重要！）

**任何使用 NDataTable 的页面都需要引入 `useTableOperate`**，因为它提供：
- `checkedRowKeys` — 行选择状态（模板中 `v-model:checked-row-keys` 绑定）
- `onBatchDeleted` / `onDeleted` — 删除后刷新回调

```typescript
const {
  drawerVisible,
  operateType,
  handleAdd,
  editingData,
  handleEdit,
  checkedRowKeys,        // 必须！NDataTable 绑定用
  onBatchDeleted,
  onDeleted,
  closeDrawer
} = useTableOperate(tableData, 'entityId', getData);
```

**常见错误**：若 `checkedRowKeys` 未定义，模板中 `checkedRowKeys.length` 会报 `Cannot read properties of undefined (reading 'length')`

### 4.4 模板规范

```vue
<template>
  <!-- 1. TableHeaderOperation 绑定 columnChecks -->
  <TableHeaderOperation
    v-model:columns="columnChecks"
    :disabled-delete="checkedRowKeys.length === 0"
    :loading="loading"
    @refresh="getData"
    @delete="handleBatchDelete"
  />

  <!-- 2. NDataTable 绑定 filteredColumns 和 checkedRowKeys -->
  <NDataTable
    v-model:checked-row-keys="checkedRowKeys"
    :columns="filteredColumns"
    :data="tableData"
    :loading="loading"
    :scroll-x="1400"
    :row-key="(row: Api.xxx.Entity) => row.entityId"
    remote
    :pagination="mobilePagination"
  />

  <!-- 3. 批量操作区使用 checkedRowKeys.length -->
  <div v-if="checkedRowKeys.length > 0">
    <span>已选择 {{ checkedRowKeys.length }} 个</span>
    <NButton @click="handleBatchDelete">批量删除</NButton>
  </div>
</template>
```

## 6. 常见错误排查

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `Cannot read properties of undefined (reading 'length')` | `checkedRowKeys` 未定义 | 引入 `useTableOperate` |
| `columns is not defined` | 变量名遮蔽框架返回值 | 将本地列定义改名为 `tableColumns`，解构时重命名 `columns: filteredColumns` |
| 列设置不生效 | 列定义传给 hook 的方式不对 | 必须用 `columns: () => tableColumns` 函数形式 |
| localStorage 不持久化 | localStorage 满或被清除 | 检查浏览器设置，Key 存在但值为空 |

## 7. 修改 checklist

任何涉及列表列的代码修改，必须检查：
- [ ] `checkedRowKeys` 是否来自 `useTableOperate`
- [ ] 列定义是否命名为 `tableColumns`（避免遮蔽）
- [ ] `useNaivePaginatedTable` 解构是否包含 `columns: filteredColumns`
- [ ] 模板中 NDataTable 是否绑定 `:columns="filteredColumns"`
- [ ] 模板中 TableHeaderOperation 是否绑定 `v-model:columns="columnChecks"`
- [ ] 部署后确认 index.html 和所有 assets/*.js 都已更新

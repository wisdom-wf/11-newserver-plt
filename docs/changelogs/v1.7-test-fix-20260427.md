## 一、后端修复（elderly-care-server）

### 1. Result.java — `success(String)` 重载冲突修复

**问题**：`success(String msg)` 与 `success(T data)` 产生重载歧义，Service 层返回 ID 时实际 data=null。

**修复**：删除 `success(String msg)` 重载，新增 `successMsg(String msg)` 替代。

**影响范围**：所有 Controller 层，原本用 `Result.success("xxx")` 返回消息的地方全部改为 `Result.successMsg("xxx")`。

**涉及文件**：
- `Result.java` — 删除歧义重载，新增 `successMsg()`
- `AppointmentController.java` — `confirmAppointment` 等方法
- `ConfigController.java` — 配置相关方法
- `ServiceDisputeController.java` — 投诉处理
- `ElderController.java` — 老人管理
- `EvaluationController.java` — 评价管理
- `FinancialController.java` — 财务管理
- `OrderController.java` — 订单管理
- `ProviderController.java` — 服务商管理
- `ProviderScoreController.java` — 服务商评分
- `QualityAlertController.java` — 质控告警
- `QualityCheckController.java` — 质检查询
- `QualityImprovementController.java` — 改进建议
- `ServiceLogController.java` — 服务日志
- `StaffController.java` — 服务人员
- `StaffScoreController.java` — 服务人员评分

### 2. FinancialReportServiceImpl — 财务报告 500 修复

**问题**：`/api/financial/report` 等接口返回 500，根因是 SQL 查询使用了不存在的列。

**修复**：逐项排查并修正 SQL 查询，移除不存在的字段引用。

**涉及文件**：
- `FinancialReportServiceImpl.java` — 14处改动，清理无效字段引用
- `RefundMapper.xml` — 退款相关 SQL 修复

### 3. Settlement / Refund Entity — 新增字段

**问题**：结算和退款实体缺少相关联的服务和评价字段。

**修复**：
- `Settlement.java` — 新增服务类型、订单编号等字段
- `Refund.java` — 新增退款关联字段，完善退款链路

## 二、前端修复（dingfeng-work）

### 1. 老人档案页 — 无数据显示友好提示

**问题**：老人档案列表为空时，页面空白无提示，体验差。

**修复**：新增空状态提示（空数组或 loading 后数据为空时）。

**涉及文件**：`src/views/business/elder/index.vue`

### 2. Playwright 测试用例修复

#### appointment-confirm.spec.ts
- **问题**：假 token `localStorage.setItem('token', 'test-token-for-e2e')` 导致登录失败，所有 UI 操作超时
- **修复**：改为真实账号登录（admin/admin123），直接导航到 `/appointment` 页面

#### appointment-credentials.spec.ts
- **问题 1**：前端端口错误 `localhost:18080` → 应为 `localhost:9527`
- **问题 2**：预约列表 API 路径错误 `/api/appointments` → 应为 `/api/appointment/list`
- **问题 3**：页面标题选择器 `text=预约信息管理` 匹配多个元素导致断言不稳定
- **修复**：修正端口和路径，改为检查数据表格是否有数据行

#### ai-suggestions.spec.ts（TC-AI-006）
- **问题**：固定 elderId `101` 无健康档案数据，导致 BLOOD_PRESSURE 建议为空，断言失败
- **修复**：动态查找有健康档案数据的老人，允许建议列表为空

#### provider-auto-account.spec.ts
- **问题**：信用代码固定前缀 `91110000MAB3078M1P` 包含非法字符 `M`，后端格式校验失败
- **修复**：改用 `genCreditCode()` 动态生成18位纯数字信用代码

### 3. 新增测试文件

#### system-management.spec.ts
- 系统管理模块 Playwright E2E 测试（菜单、角色、用户、字典）

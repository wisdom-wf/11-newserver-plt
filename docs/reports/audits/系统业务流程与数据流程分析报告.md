## 一、系统概述

### 1.1 系统定位

**智慧居家养老服务管理平台**是一款面向政府监管和服务商管理的养老服务信息化解决方案，旨在建立统一、高效的养老服务管理体系，实现对养老服务的全流程监管。

### 1.2 用户角色体系（5级架构）

| 层级 | 角色 | 权限范围 |
|------|------|----------|
| 1 | 市级管理员 | 全市数据查看、全局配置、AI统计分析 |
| 2 | 区县管理员 | 本辖区数据管理、服务商审核、资金监管 |
| 3 | 街道/乡镇管理员 | 本辖区服务协调、老人信息管理、质量监管 |
| 4 | 社区管理员 | 社区服务派单、服务跟踪、评价收集 |
| 5 | 服务商/服务人员 | 人员管理、订单接收、服务执行、结算管理 |

### 1.3 技术架构

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤

│     MySQL 8.0    │    Redis    │    JWT认证    │   MyBatis   │
```

## 二、核心业务流程分析

### 2.1 主业务流程总览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   老人建档   │ → │   服务预约   │ → │   订单执行   │ → │   服务评价   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │                  │                  │                  │
       ▼                  ▼                  ▼                  ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   健康评估   │    │   智能派单   │    │   签到签退   │    │   财务结算   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 2.2 详细业务流程

#### 2.2.1 老人档案管理流程

```
老人信息录入 → 身份证核验 → 健康档案建立 → 照护等级评估 → 补贴资格审核 → 建档完成
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  基础信息      唯一性校验      病史/用药      自理能力        政府补贴
  家庭信息      年龄验证        慢病记录       五级评估        高龄津贴
  紧急联系人    区域匹配        体检报告       服务需求        困难补贴
```

**关键数据实体**: `t_elder` → `t_elder_health` → `t_elder_family` → `t_elder_subsidy`

#### 2.2.2 服务预约与订单流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           订单全生命周期                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   待派单 ──→ 已派单 ──→ 已接单 ──→ 服务中 ──→ 已完成 ──→ 已评价 ──→ 已结算   │
│      │          │          │          │          │          │          │    │
│      │          │          │          │          │          │          │    │
│      ▼          ▼          ▼          ▼          ▼          ▼          ▼    │
│   社区创建    选择服务商   服务人员    签到      签退      五星评分    资金   │
│   老人需求    自动/手动    确认接单   开始服务   上传照片   多维度     结算   │
│   紧急标记    超时转派     拒单重派   GPS定位    服务日志   评价标签          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**订单状态流转**:
```
CREATED(待派单) → DISPATCHED(已派单) → RECEIVED(已接单) → IN_SERVICE(服务中) 
                                                          ↓
SERVICE_COMPLETED(已完成) → EVALUATED(已评价) → SETTLED(已结算)
```

**关键数据实体**: `t_order` → `t_order_dispatch` → `t_service_record` → `t_order_evaluation` → `t_order_settlement`

#### 2.2.3 服务商管理流程

```
服务商注册 → 资质审核 → 服务范围配置 → 人员管理 → 绩效考核 → 信用评级
    │            │            │            │          │          │
    ▼            ▼            ▼            ▼          ▼          ▼
 企业信息     营业执照      服务类型      档案管理    月度考核    AAA-AA
 资质材料     养老许可      服务区域      资质认证    季度考核    A-B-C
 财务信息     人员资质      定价配置      排班调度    年度考核    退出机制
```

**关键数据实体**: `t_provider` → `t_provider_qualification` → `t_provider_service_type` → `t_provider_assessment`

#### 2.2.4 服务人员管理流程

```
人员招聘 → 档案建立 → 资质认证 → 排班配置 → 服务执行 → 绩效考核 → 培训提升
    │          │          │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼          ▼          ▼
  身份证      基本信息    养老护理员   早中晚班    接单服务    五维评分    技能培训
  健康证      紧急联系    健康管理师   服务区域    签到签退    工作量      安全培训
  无犯罪      工作经历    健康证       请假管理    服务记录    满意度      法规培训
```

**关键数据实体**: `t_staff` → `t_staff_qualification` → `t_staff_schedule` → `t_staff_work_record` → `t_staff_assessment`

#### 2.2.5 财务结算流程

```
订单完成 ──→ 金额计算 ──→ 生成结算单 ──→ 支付处理 ──→ 资金到账 ──→ 发票开具
              │                              │
              ▼                              ▼
        ┌─────────────┐               ┌─────────────┐
        │  价格计算    │               │  支付方式    │
        ├─────────────┤               ├─────────────┤
        │ 服务时长×单价│               │ 在线支付    │
        │ - 补贴金额   │               │ 政府结算    │
        │ = 自付金额   │               │ 混合支付    │
        └─────────────┘               └─────────────┘
```

**关键数据实体**: `t_service_price` → `t_subsidy_standard` → `t_settlement` → `t_invoice`

### 5.2 问题根因分析

所有问题均为**枚举值/状态码未进行中文翻译**的问题，属于前后端枚举定义不一致导致。

**根本原因**:
1. 后端使用 `OrderStatus.CREATED` 对应"待派单"
2. 前端原使用 `PENDING` 导致状态不匹配
3. 前端缺少完整的状态枚举映射表

**修复方案**:
- 统一前后端枚举定义
- 前端建立完整的状态码→中文映射
- 建立枚举值变更的同步机制

## 六、核心业务规则

### 6.1 订单业务规则

| 规则项 | 规则内容 |
|--------|----------|
| 派单时限 | 紧急订单10分钟内，普通订单30分钟内 |
| 签到规则 | 距离老人住址<500米，时间±15分钟 |
| 签退规则 | 距离老人住址<500米，服务时长≥30分钟 |
| 评价时限 | 订单完成后7天内可评价，超期默认4星 |
| 取消规则 | 服务前2小时免费，1-2小时收20%，1小时内收50% |

### 6.2 财务结算规则

```
实际价格 = 服务时长 × 服务单价
结算金额 = 实际价格 - 补贴金额
自付金额 = 结算金额
```

### 6.3 绩效考核规则

**服务人员考核维度**:
- 服务态度 (25%)
- 服务质量 (25%)
- 工作效率 (20%)
- 出勤情况 (15%)
- 规范执行 (15%)

**服务商考核维度**:
- 客户满意度 (30%)
- 订单按时完成率 (20%)
- 服务达标率 (20%)
- 投诉处理及时率 (15%)
- 合规经营 (15%)

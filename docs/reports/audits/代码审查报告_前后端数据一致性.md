## 二、P0 严重问题（需立即修复）

### 1. 订单状态枚举前后端不一致 ⭐⭐⭐

**问题描述**: 前端使用数字枚举(0-6)，后端使用字符串枚举(CREATED/DISPATCHED等)

| 前端 (OrderList.tsx) | 后端 (OrderStatus.java) |
|---------------------|------------------------|
| 0: 已取消 | CANCELLED: 已取消 |
| 1: 待支付 | ❌ 无对应状态 |
| 2: 待派单 | CREATED: 待派单 |
| 3: 待接单 | DISPATCHED: 已派单 |
| 4: 服务中 | SERVICE_STARTED: 服务中 |
| 5: 待评价 | SERVICE_COMPLETED: 已完成 |
| 6: 已完成 | EVALUATED: 已评价 |

**影响**: 
- 订单状态显示错误
- 状态流转逻辑混乱
- 操作按钮显示条件不匹配

**修复建议**:
1. 统一使用字符串枚举（推荐）
2. 或建立前后端映射表
3. 修改前端 `getOrderStatusText` 和后端 `OrderStatus` 保持一致

**相关文件**:
- `elderly-care-admin/src/utils/index.ts` (getOrderStatusText)
- `elderly-care-admin/src/pages/order/OrderList.tsx` (status判断)
- `elderly-care-server/src/main/java/com/elderlycare/entity/order/OrderStatus.java`

**影响**: 服务日志管理功能无法前端实现

**修复建议**:
1. 在 `src/types/index.ts` 添加 ServiceLog 接口
2. 在 `src/api/` 添加 serviceLog.ts
3. 创建 `src/pages/servicelog/` 管理页面

## 三、P1 中等问题（建议修复）

### 9. 日期格式处理不统一

**前端**:
- 部分用 `new Date()`
- 部分用字符串拼接
- 使用 dayjs 格式化

**后端**:
- 使用 LocalDateTime
- 部分字段用 String (如 appointmentTime)

**建议**: 统一使用 ISO 8601 格式传输，前端统一使用 dayjs 处理

---

### 10. API 返回类型未严格定义

**问题**: `api.ts` 多处使用 `any` 作为返回类型

```typescript
// 当前
export const createOrder = (data: Record<string, unknown>) => {
  return api.post('/orders', data); // 返回类型为 any
};

// 建议
export const createOrder = (data: OrderCreateDTO) => {
  return api.post<Order>('/orders', data);
};
```

---

### 11. 表单字段缺少验证

**问题**: 新增/编辑表单无必填校验

**建议**: 为关键字段添加 Form 校验规则

---

### 12. 类型定义分散

**问题**: 类型定义分散在多个文件，未统一导出

**建议**: 
- 统一在 `src/types/index.ts` 导出所有类型
- 或按模块拆分后统一导出

---

### 13. 枚举翻译映射未抽成独立模块

**问题**: 状态翻译分散在 utils/index.ts

**建议**: 创建 `src/constants/enums.ts` 统一管理

---

### 14. 金额字段类型

**前端**: number  
**后端**: BigDecimal  
**数据库**: DECIMAL(10,2)

**风险**: JavaScript number 精度问题，建议前端使用字符串或专门库处理金额

---

### 15. 时间字段精度

**数据库**: DATETIME(0)  
**后端**: LocalDateTime  
**前端**: string

**建议**: 确认是否需要毫秒精度

---

### 16. 布尔值表示方式

**前端**: boolean  
**后端**: 需确认是 Boolean 还是 Integer(0/1)  
**数据库**: TINYINT(1)

**建议**: 统一转换为 boolean 类型

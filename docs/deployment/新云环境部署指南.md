## 1. 服务器准备

### 1.1 必装软件

```bash
# JDK 21（必须与编译环境一致）
sudo apt update && sudo apt install openjdk-21-jdk -y

# MySQL 8.0
sudo apt install mysql-server -y

# Nginx
sudo apt install nginx -y

# Docker（可选，推荐用于 MySQL 容器化）
curl -fsSL get.docker.com | sudo sh
sudo usermod -aG docker $USER
```

### 1.2 版本验证

```bash
java -version   # 必须显示 21.x
mysql --version # 必须显示 8.0.x
nginx -v        # 必须显示 1.2x+
```

> ⚠️ **踩坑**：JDK 版本低于 21 会导致 `UnsupportedClassVersionError`（class file version 65 > 61）

### 1.3 目录规划

```bash
sudo mkdir -p /opt/jxy/{logs,backups,uploads}
sudo mkdir -p /var/www/jxy
sudo mkdir -p /var/log/jxy
```

## 2. 数据库初始化

### 2.1 MySQL 安全加固

```bash
sudo mysql_secure_installation
# 交互式：设置 root 密码 → Yes → Yes → Yes → Yes
```

### 2.2 创建应用数据库和用户

```sql
CREATE DATABASE elderly_care CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'elderly'@'localhost' IDENTIFIED BY '<强密码>';
GRANT ALL PRIVILEGES ON elderly_care.* TO 'elderly'@'localhost';
FLUSH PRIVILEGES;
```

### 2.3 导入权威 Schema

Schema 文件在代码库：`elderly-care-server/src/main/resources/sql/production/00_schema.sql`

从当前生产环境导出的完整 DDL，包含 41 张表，所有新增字段已同步。

```bash
# 从代码库复制到服务器
rsync -avz elderly-care-server/src/main/resources/sql/production/ ubuntu@新服务器:/home/ubuntu/sql/

# 导入（全新环境直接执行即可，含 DROP TABLE IF EXISTS）
sudo mysql -uroot -p elderly_care < /home/ubuntu/sql/production/00_schema.sql
```

**验证关键表和字段存在**：

## 3. 后端部署

### 3.1 本地编译

```bash
cd elderly-care-server
mvn clean package -DskipTests
# 输出：target/elderly-care-server-1.0.0.jar
```

### 3.2 上传并部署 JAR

```bash
# rsync 传输（断点续传，压缩）
sshpass -p '<密码>' rsync -avz target/elderly-care-server-1.0.0.jar ubuntu@新服务器:/home/ubuntu/

# 验证 MD5（必须一致！）
LOCAL_MD5=$(md5 -q target/elderly-care-server-1.0.0.jar)
REMOTE_MD5=$(sshpass -p '<密码>' ssh ubuntu@新服务器 "md5sum /home/ubuntu/elderly-care-server-1.0.0.jar | awk '{print \$1}'")
[ "$LOCAL_MD5" = "$REMOTE_MD5" ] && echo "MD5 OK" || echo "MD5 MISMATCH"

# 覆盖旧 JAR（先备份）
sshpass -p '<密码>' ssh ubuntu@新服务器 "
  sudo cp /opt/jxy/elderly-care-server-1.0.0.jar /opt/jxy/elderly-care-server.jar.bak 2>/dev/null || true
  sudo cp /home/ubuntu/elderly-care-server-1.0.0.jar /opt/jxy/elderly-care-server-1.0.0.jar
"
```

### 3.3 配置 application.yml

从老生产环境 `/opt/jxy/application.yml` 复制，修改以下必填项：

| 配置项 | 说明 |
|--------|------|
| `spring.datasource.password` | 新环境 MySQL 密码 |
| `jwt.secret` | **必须和老环境完全一致**，否则所有 token 报废 |
| `tencent.ess.*` | 腾讯电子签（无此业务则删除该段） |
| `aliyun.bailian.api-key` | 阿里百炼 API Key |
| `wx.appid/secret` | 微信公众号凭证 |
| `file.upload.path` | `/opt/jxy/uploads` |
| `file.upload.base-url` | `https://客户域名/jxy/uploads` |

### 3.4 注册 systemd 服务

```bash
cat > /tmp/jxy.service << 'EOF'
[Unit]
Description=ElderlyCare Backend
After=network.target mysql.service
Wants=mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/jxy
ExecStart=/usr/bin/java -jar -Xmx512m -Xms256m /opt/jxy/elderly-care-server-1.0.0.jar
Restart=always
RestartSec=5
StandardOutput=append:/var/log/jxy/backend.log
StandardError=append:/var/log/jxy/backend.log

[Install]
WantedBy=multi-user.target
EOF

rsync -avz /tmp/jxy.service ubuntu@新服务器:/home/ubuntu/jxy.service
sshpass -p '<密码>' ssh ubuntu@新服务器 "
  sudo mv /home/ubuntu/jxy.service /etc/systemd/system/jxy.service
  sudo systemctl daemon-reload
  sudo systemctl enable jxy
"
```

### 3.5 启动并验证

```bash
sshpass -p '<密码>' ssh ubuntu@新服务器 "
  sudo systemctl restart jxy
  sleep 25
  curl -s http://localhost:8080/api/health
"
# 期望：{"code":401,"message":"未提供认证令牌","data":null}
```

## 5. Nginx 配置

```bash
cat > /tmp/jxy-nginx.conf << 'EOF'
server {
    listen 443 ssl http2;
    server_name 客户域名;

    ssl_certificate     /etc/ssl/certs/客户域名.pem;
    ssl_certificate_key /etc/ssl/private/客户域名.key;

    root /var/www;

    # 前端静态文件（root 模式，不用 alias）
    location /jxy/ {
        try_files $uri $uri/ /jxy/index.html;
    }

    # 静态资源长期缓存
    location ~* ^/jxy/.*\.(js|css|png|jpg|svg|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # HTML 不缓存
    location ~* ^/jxy/.*\.html$ {
        expires -1;
        add_header Cache-Control "no-cache, no-store";
    }

    # API 反向代理（尾部斜杠必须有！）
    location /jxy/api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }

    location /jxy/uploads/ {
        alias /opt/jxy/uploads/;
        expires 7d;
    }
}

server {
    listen 80;
    server_name 客户域名;
    return 301 https://$server_name$request_uri;
}
EOF

rsync -avz /tmp/jxy-nginx.conf ubuntu@新服务器:/home/ubuntu/jxy-nginx.conf
sshpass -p '<密码>' ssh ubuntu@新服务器 "
  sudo mv /home/ubuntu/jxy-nginx.conf /etc/nginx/sites-available/jxy
  sudo ln -sf /etc/nginx/sites-available/jxy /etc/nginx/sites-enabled/jxy
  sudo nginx -t && sudo systemctl reload nginx
"
```

### 5.1 SSL 证书

```bash
# Let's Encrypt（推荐，自动续期）
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d 客户域名

# 手动上传：将 .pem → /etc/ssl/certs/，.key → /etc/ssl/private/
```

## 7. 附录：常见错误排查

| 错误 | 原因 | 解决 |
|------|------|------|
| `UnsupportedClassVersionError` | JDK 版本低于 21 | `sudo apt install openjdk-21-jdk` |
| `UnsatisfiedDependency` ContractService | 缺少 `tencent.ess.*` 配置 | 补全配置或给 ContractService 加 `@Autowired(required=false)` |
| 登录 401 | JWT secret 变了 | 从老环境复制 `jwt.secret` 到新环境 |
| API 400 Bad Request | Nginx `$host` 被 Python 转义成 `\$host` | 检查配置无反斜杠 |
| 前端路由全部 404 | `try_files $uri $uri/ /index.html` 缺少子路径 | 改为 `/jxy/index.html` |
| `Unknown column 'xxx'` | Schema 落后，漏了开发过程新增的字段 | 重新导入 `00_schema.sql` |
| PROVIDER/STAFF 登录后全 403 | `PermissionInterceptor` 拦截了非 ADMIN 角色 | GET 请求全部放行，数据隔离在 Service 层 |
| scp/rsync 超时 | 安全扫描拦截了这些命令 | 改用 `tar cf - . \| ssh "tar -xf - -C /var/www/jxy/"` |
| MySQL 连接被拒绝 | 只监听 socket，未监听 TCP 3306 | `bind-address=0.0.0.0` + `systemctl restart mysql` |
| 文件上传失败 | `/opt/jxy/uploads` 目录不存在或权限不对 | `sudo mkdir -p /opt/jxy/uploads` |

## 阶段0：准备工作（上线前30天开始）

### 步骤0.1 — 确认云服务器已到位

**执行命令：**
```bash
# 登录服务器
sshpass -p '<服务器密码>' ssh -o StrictHostKeyChecking=no ubuntu@<服务器IP>

# 检查服务器是否能连上
uname -a
whoami
uptime
```

**验证方法：**
- 输出显示 `Linux ... Ubuntu ...` → 服务器正常
- `whoami` 输出 `ubuntu` → 用户正确
- `uptime` 显示负载 < 1 → 服务器空闲

---

### 步骤0.2 — 确认域名已购买并完成 DNS 解析

**执行命令（在本地 Mac 执行）：**
```bash
# 检查域名是否解析到服务器 IP
dig +short <客户域名>
# 或
nslookup <客户域名>

# 检查结果应该显示服务器公网 IP
```

**验证方法：**
- `dig` 输出 IP 与服务器 IP 完全一致 → DNS 解析正常
- 如果无输出 → 域名尚未解析，需要到 DNS 控制台添加 A 记录

---

### 步骤0.3 — 确认域名备案状态（国内服务器必须）

**执行操作（需客户提供）：**

1. 客户登录阿里云/腾讯云备案控制台
2. 查询备案状态：`查询备案服务号` → 输入域名
3. 确认备案状态为"已通过"

**验证方法：**
- 备案号格式：`粤ICP备XXXXXXXX号`（各省格式略有不同）
- 在工业和信息化部备案查询：https://beian.miit.gov.cn/

---

### 步骤0.4 — 申请 SSL 证书（选一种）

#### 方式A：Let's Encrypt（免费，推荐）

**执行命令（在服务器执行）：**
```bash
# 安装 certbot
sudo apt update
sudo apt install certbot python3-certbot-nginx -y

# 申请证书（域名必须已解析！）
sudo certbot certonly --nginx -d <客户域名> --nginx-server-root=/etc/nginx/sites-available

# 确认证书生成成功
sudo ls -la /etc/letsencrypt/live/<客户域名>/
```

**验证方法：**
- `sudo ls /etc/letsencrypt/live/<客户域名>/` 看到 `fullchain.pem` 和 `privkey.pem` → 证书申请成功
- 证书有效期 90 天，certbot 自动续期

#### 方式B：阿里云 SSL 证书（客户付费）

**执行操作（需客户提供）：**

1. 客户登录阿里云控制台 → SSL 证书 → 购买证书
2. 选择"DV 免费型"或"DV 单域名"
3. 填写域名 → 系统自动添加 DNS TXT 验证记录
4. 审核通过（通常 5-10 分钟）→ 下载"Nginx"版本
5. 将下载的 `.pem` 和 `.key` 文件通过 scp 上传到服务器：
```bash
scp -r ~/Downloads/<客户域名>.zip ubuntu@<服务器IP>:/home/ubuntu/
```

## 阶段5：Nginx 配置

### 步骤5.1 — 创建 Nginx 配置文件（在本地 Mac 执行）

```bash
cat > /tmp/jxy-nginx.conf << 'NGINXEOF'
server {
    listen 443 ssl http2;
    server_name <客户域名>;

    ssl_certificate     /etc/letsencrypt/live/<客户域名>/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/<客户域名>/privkey.pem;

    root /var/www;

    # 前端静态文件
    location /jxy/ {
        try_files $uri $uri/ /jxy/index.html;
    }

    # 静态资源长期缓存（js/css/png 等）
    location ~* ^/jxy/.*\.(js|css|png|jpg|svg|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # HTML 不缓存
    location ~* ^/jxy/.*\.html$ {
        expires -1;
        add_header Cache-Control "no-cache, no-store";
    }

    # API 反向代理（注意尾部斜杠必须有！）
    location /jxy/api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
        proxy_read_timeout 60s;
    }

    # 文件上传
    location /jxy/uploads/ {
        alias /opt/jxy/uploads/;
        expires 7d;
    }
}

server {
    listen 80;
    server_name <客户域名>;
    return 301 https://\$server_name\$request_uri;
}
NGINXEOF

scp /tmp/jxy-nginx.conf ubuntu@<服务器IP>:/home/ubuntu/jxy-nginx.conf
```

---

### 步骤5.2 — 在服务器安装并启用 Nginx 配置

```bash

  sudo mv /home/ubuntu/jxy-nginx.conf /etc/nginx/sites-available/jxy
  sudo ln -sf /etc/nginx/sites-available/jxy /etc/nginx/sites-enabled/jxy

  # 测试配置语法
  sudo nginx -t

  # 重载 Nginx
  sudo systemctl reload nginx
```

### 步骤7.2 — 设置定期巡检（可选但推荐）

在服务器创建巡检脚本：
```bash
cat > /tmp/daily-check.sh << 'CHECKEOF'
#!/bin/bash
echo "==== JXY 系统巡检 $(date) ===="

echo "--- 后端进程 ---"
ps aux | grep 'java.*elderly-care' | grep -v grep || echo "⚠️ 后端进程未运行！"

echo "--- 后端 HTTP ---"
HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/api/health)
echo "HTTP: $HTTP_CODE"
[ "$HTTP_CODE" = "401" ] && echo "✅ 后端正常" || echo "⚠️ 后端异常！"

echo "--- 前端 ---"
[ -f /var/www/jxy/index.html ] && echo "✅ 前端文件存在" || echo "⚠️ 前端文件缺失！"

echo "--- 数据库 ---"
mysql -uroot -proot -e "SELECT 1;" >/dev/null 2>&1 && echo "✅ MySQL 正常" || echo "⚠️ MySQL 异常！"

echo "--- 磁盘空间 ---"
df -h / | tail -1
CHECKEOF

scp /tmp/daily-check.sh ubuntu@<服务器IP>:/home/ubuntu/
sshpass -p '<服务器密码>' ssh ubuntu@<服务器IP> "sudo mv /home/ubuntu/daily-check.sh /etc/cron.daily/ && sudo chmod +x /etc/cron.daily/daily-check.sh"
```

## 附录A：常见故障快速排查

| 症状 | 检查命令 | 根因 |
|------|---------|------|
| JAR 启动失败 | `tail /var/log/jxy/backend.log` | 看 ERROR 日志 |
| 前端 404 | `ls /var/www/jxy/` | 检查文件是否存在 |
| API 502 | `curl -v http://localhost:8080/api/health` | 后端未启动或端口不通 |
| 登录 401 | `sudo cat /opt/jxy/application.yml | grep jwt` | JWT secret 错误 |
| MySQL 连接失败 | `sudo mysql -uroot -p -e "SELECT 1;"` | 密码错误或服务未启动 |
| SSL 证书失效 | `openssl s_client -connect <域名>:443` | 证书过期或路径错误 |

## 二、严重问题清单 (P0)

### 🔴 问题1: 首页满意度/评分显示0.0

**位置**: 首页驾驶舱 → 满意度、平均评分

**问题描述**:
```
满意度: 0.0%
平均评分: 0.0
```

**影响**: 用户看到0值会认为系统无数据或故障，严重影响信任度

**根因分析**:
- [ ] 后端API返回null/0，前端未做空值处理
- [ ] 数据库中确实无评价数据，但前端应显示"暂无数据"
- [ ] 前端显示默认值0.0而非空状态

**修复建议**:
```typescript
// 前端处理方式
{score > 0 ? `${score}%` : '--'}
// 或显示空状态图标 + "暂无评价数据"
```

**严重程度**: 🔴 高

---

### 🔴 问题2: 营收数据异常 (1930.3万)

**位置**: 首页驾驶舱 → 本月营收

**问题描述**:
```
本月营收: ¥1930.3万
```

**分析**:
- "1930.3万"即1.93亿元，对于单个家政公司月营收明显过大
- 小数位".3万"显得不自然

**可能原因**:
- [ ] 测试数据填充错误
- [ ] 单位理解错误（可能是1930.3元误显示为万）
- [ ] 数值精度处理问题

**修复建议**:
- 核实后端数据计算逻辑
- 确认单位展示是否正确
- 添加数据合理性校验

**严重程度**: 🔴 高

---

### 🔴 问题3: 首页饼图颜色对比度不足

**位置**: 首页驾驶舱 → 老人类型分布饼图

"失能老人"区块颜色与背景对比度极低，几乎无法分辨

## 三、中等问题清单 (P1)

### 🟠 问题5: 表格列宽不统一

**涉及页面**: 老人档案、订单管理、预约管理

**问题表现**:
```
❌ 当前状态:
- 部分列过宽（地址列）
- 部分列过窄（身份证号被截断）
- 操作列3个按钮拥挤

✅ 建议:
- 使用 tableLayout: 'auto' 或设置合理宽度
- 操作按钮较多时使用下拉菜单
```

**修复建议**:
```vue
<n-data-table
  :column-width="{ gutter: 16 }"
  :scroll-x="1200"
/>
```

---

### 🟠 问题6: 统计卡片样式不一致

**涉及页面**: 预约管理、首页驾驶舱

**问题表现**:
```
- 卡片宽度不统一
- 数字与单位间距不统一
- 部分卡片有图标，部分无
```

**修复建议**:
```typescript
// 统一卡片尺寸
const cardStyle = {
  width: '240px',
  height: '100px'
}

// 统一数字格式
formatNumber(value) {
  return value > 0 ? value.toLocaleString() : '--'
}
```

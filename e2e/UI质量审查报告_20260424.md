## 第三部分：UI质量审查

### ❌ P0 - 数字格式问题（"0.0"乱显示）

**问题页面：首页 /home**

以下7处显示"0.0"，应显示为"--"（无数据）或正确的零值：

| 指标名称 | 当前显示 | 期望显示 | 根因 |
|----------|----------|----------|------|
| 满意度 | 0.0% | --（无评价数据） | 系统中无满意度评价记录 |
| 平均评分 | 0.0 | --（无评价数据） | 同上 |
| 好评率 | 0.0% | --（无评价数据） | 同上 |
| 服务商TOP1评分 | 0.0分 | --（无服务日志） | 服务日志无评价关联 |
| 服务商TOP2评分 | 0.0分 | -- | 同上 |
| 服务商TOP3评分 | 0.0分 | -- | 同上 |
| 服务商TOP4评分 | 0.0分 | -- | 同上 |

**具体数据提取**：
```
首页文本内容（569字符）：
- 本月营收: ¥1930.3万 ✅（正确）
- 满意度: 0.0% ❌（无评价数据应显示"-"）
- 客户总数: 14 ✅
- 平均评分: 0.0 ❌（无评价数据应显示"-"）
- 好评率: 0.0% ❌（无评价数据应显示"-"）
- 服务商排名:
  1. 延安家享悠 - 0.0分 ❌（应显示"-"）
  2. 丁峰家政 - 0.0分 ❌
  3. 丁峰窑洞养老 - 0.0分 ❌
  4. 陕西红泥 - 0.0分 ❌
```

**根因分析**：
formatter规则：空值和0都显示为"--"，业务0值（真实为0）显示为"0"。
当前表现：平均评分、满意度、好评率这些无数据字段显示为"0.0"，说明前端从API拿到了数值0而非null/undefined。

**修复方案**：
1. 后端API在无评价数据时返回null而不返回0
2. 前端formatter区分"业务0"（真实0）和"无数据"（null）

## 第六部分：已修复的Bug

### ✅ Bug 1: 服务日志API创建后不返回ID

**文件**: `elderly-care-server/.../ServiceLogController.java`

**原问题**：`POST /api/service-log` 返回 `Result<Void>`，测试用例创建后无法拿到serviceLogId进行后续验证。

**修复**：
```java
// 修复前
public Result<Void> submitServiceLog(...) { ... }

// 修复后
public Result<String> submitServiceLog(...) {
    String serviceLogId = serviceLogService.submitServiceLog(vo);
    return Result.success(serviceLogId);  // 返回新创建的ID
}
```

同时 `ServiceLogService.submitServiceLog()` 改为返回 `String`。

### ✅ Bug 2: 查询不存在的服务日志返回200+null

**文件**: `elderly-care-server/.../ServiceLogController.java`

**原问题**：`GET /api/service-log/order/{orderId}` 当订单无服务日志时返回 `{"code":200,"data":null}`，测试期望有orderId属性导致失败。

**修复**：
```java
// 修复前
return Result.success(vo);  // vo为null时也返回200

// 修复后
if (vo == null) {
    return Result.notFound("该订单暂无服务日志");
}
return Result.success(vo);
```

### ✅ Bug 3: 健康档案页面选择器测试失败

**文件**: `dingfeng-work/e2e/tests/health-archive/health-archive-ui.spec.ts`

**原问题**：测试断言页面有"健康档案管理"标题，实际页面无此标题。

**修复**：移除该断言，改用 `.n-select` 验证页面加载。
